# DESIGN.md — UX-решения по экранам

Источник истины по структуре экранов. Визуальные токены — в `DESIGN-SYSTEM.md`
(подчиняется правилам раздела «Дизайн» в CLAUDE.md). Утверждено по итогам
этапа B0 (прототип: `docs/prototypes/lenta-b0.html`).

---

## 1. Каркас приложения

- Левый сайдбар: логотип VSMS; навигация — Лента отгрузок, Планировщик,
  Приёмка, Контракты, Тара, Ингредиенты, Логистика материалов, Аналитика,
  Справочники, Настройки. Активный пункт — заливка `canvas-soft-2`.
- Контент: заголовок страницы + подзаголовок-контекст
  («Овощное сырьё на завод · сезон 2026»).
- Контейнер контента: fluid, ширина 100% минус паддинги 24–32px,
  max-width 1880px, центрирование на более широких экранах.

## 2. Лента отгрузок (главный экран)

### Тулбар
- «+ Отгрузка» (primary, чёрная) → диалог «Отгрузка» (раздел 4).
- Переключатель недели ‹ Неделя N даты › + кнопка «Сегодня».
- Поиск: фермер, культура, № акта.
- Фильтры: Поставщик, Сырьё, Статус (combobox с поиском).
- Тумблер «Скрыть плановые». Экспорт Excel.
- Переключатель видов: Таблица | Heatmap | План (B3 — только Таблица,
  остальные — заглушки/скрыты до своих этапов).

### Группировка
- Недели ISO (BR-17). Шапка недели: чип `W{N}` (mono) + «Неделя N ·
  даты · год (Пн–Сб|Пн–Пт по сезону)» + чипы культур с недельными
  суммами (т) + справа «N машин · X т». Прошлые недели свёрнуты
  (разворачиваются кликом), текущая и будущие развёрнуты.
  При входе — скролл к текущей неделе.
- Дни (по дате ПРИБЫТИЯ): шапка «День недели, дата» + чипы культур
  дня + справа «тара: N ящиков · M бочек». Пустой день — одна строка
  «— нет отгрузок».

### Машина (двухзонный леджер)
Article-блок, машина = блок строк. Код машины НЕ показывается
(опознание: водитель/груз/даты).

**Левая зона** (фон = цвет статуса, фикс. ширина — вмещает
«30 сентября → 2 октября» с зазором до карандаша ≥8px):
- Строка 1: бейдж статуса · даты «{отправление} → {прибытие}»
  (полные месяцы, акцент weight 600 на прибытии, tabular-nums) ·
  карандаш у правого края (один на машину → диалог «Отгрузка»).
- Строка 2: «Фамилия И. О. · ТК» кликабельно → модалка водителя;
  без водителя — «водитель не назначен» приглушённо.
- Комментарий (если есть): нижней строкой под пунктирным разделителем.

**Правая зона** — строка на позицию (1–4), строки делят высоту машины
поровну (flex 1, контент по вертикальному центру), подложка = Culture.color
8–10% opacity на всю высоту строки. Без шапки колонок, без карандашей
на строках. Колонки:
- чип-квадрат цвета культуры + название;
- вес: tabular-nums + «кг» приглушённо;
- фермер;
- тара: «{тип} · {N} шт» — информационно, расчёт ceil по нормам
  (DOMAIN раздел 3); культура без тары — пусто;
- № акта: Geist Mono в рамке; наличие номера = позиция принята;
  до приёмки — «— приёмка» / «— не отправлена» приглушённо.

**Цвета статусов:** planned серый (#888/фон #f5f5f5) · sent #0070f3
(фон #d3e5ff) · arrived #f5a623 (фон #ffefcf) · accepted #29bc9b
(фон #aaffec приглушённый).

### Модалка водителя
ФИО (заголовок), бейдж ТК, телефон — `tel:`-ссылка, инфо-блок (машина,
свободный текст), кнопки «Скопировать телефон» / «Скопировать всё»
(не переполняют футер).

## 3. Действия со статусами (из B2)
- planned: «Отправить» (admin) → confirm-диалог с предпросмотром
  движений тары («Будет списано у фермеров: …»); при отсутствии нормы
  box-культуры — ошибка в диалоге, подтверждение disabled.
- sent: «Откатить в план» (admin) → confirm с текстом про сторно.
- Правка/удаление — только planned. Карандаш у sent+: для admin
  открывает диалог с заблокированными полями и подсказкой «для правки
  откатите статус», для остальных скрыт.

## 4. Диалог «Отгрузка» (создание = редактирование, одна форма)
- Заголовок «Отгрузка» (+ статус-бейдж при редактировании).
- Даты: «Отправление» и «Прибытие» рядом; хинт: «введите одну дату —
  вторая подставится (±2 дня), обе можно править». Валидация BR-11/BR-12
  серверная, ошибки под полями.
- Водитель: combobox с поиском по фамилии, опции «Фамилия И. О. · ТК»,
  необязателен (обязателен при отправке — проверка перехода).
- Комментарий: textarea 2 строки.
- Позиции (1–4): фермер (select) · культура (select) · вес кг ·
  строка контракта (select «label · цена ₽/кг»; опции = строки контрактов
  фермера+культуры сезона; ровно одна → автоподстановка, редактируема) ·
  тара — информационная строка «Тара: ≈ N {ед} (расчёт по норме)»,
  НЕ поле ввода · крестик удаления. «+ позиция» (disabled при 4).
- Футер: Отмена / Сохранить (primary).

## 5. Вне скоупа (решено НЕ делать)
Индикаторы качества, PDF, «догруз по пути», код машины в UI,
чекбоксы актов, шапка колонок позиций, карандаши на строках позиций.

## 6. Отложено
Мобильная версия ленты — отдельный дизайн-промпт после стабилизации
десктопа в коде (после B3). Виды Heatmap и План — этапы B4/B5.
