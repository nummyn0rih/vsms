# ROADMAP-C — Этап C: полная приёмка (акт партии)

Дополняет TASKS.md/DOMAIN.md. Этап C замыкает контур поставки: партия (позиция машины) принимается
актом → считается принятый вес → ингредиент приходит на склад → появляется стоимость.

## Контекст (наследуется, уже в DOMAIN)
- Приёмка **ПОЗИЦИОННАЯ** (BR-13): единица = `ShipmentItem`, акт `AcceptanceAct` 1:1, акты позиций
  одной машины — в разное время. `accepted` машины — авто (все позиции приняты); «Частично принята» —
  производный UI-статус.
- Перевеска `actual_weight_kg` вводится в B4b (инлайн) ИЛИ в форме акта — одно поле (BR-25).
- Зоны «Приёмки» (BR-26): 1 `sent` · 2 `arrived` не все приняты (вкл. частично) · 3 все приняты.
- Три базы веса (§1): принятый = факт − непринятое; идёт в контракты/стоимость.

## Решённые развилки (зафиксированы)
1. **Брак — один % на позицию** (простые культуры). Без разбивки по причинам.
2. **Калибры — только у культур со схемой калибровки** (A5). Нет схемы → блока калибров в форме нет.
3. **Принятый вес — производный**, не вводится руками. Простые: `accepted = actual × (1 − brak%)`.
   Калибровочные (огурец): через категории схемы — `accepted = actual × Σ(% категорий is_accepted=true)`
   (нестандарт >12 см — категория `is_accepted=false`, BR-10).
4. **Движение ингредиента — на позицию** (культура → ингредиент по рецептуре A-этапа), фактический вес.
5. **Акт — на позицию**; частичная приёмка машины — штатный поток.
6. **Откат приёмки позиции** — admin, сторно её движений ингредиентов, ChangeLog, идемпотентно.

## Развилка брак/калибры — РЕШЕНО (Вариант 2)
Калибровочная культура (огурец) — **двухступенчато** (BR-10 обновлён):
1. `brak_percent` (порча) — % от actual; годное = actual × (1 − brak%).
2. Калибровка ГОДНОГО по размерным категориям; нестандарт >12 см — категория `is_accepted=false`.
`accepted = actual × (1 − brak%) × Σ(% принятых размерных категорий)`. Поле «% брака» в форме акта
показывается И у simple, И у калибровочных культур. У калибровочных дополнительно — таблица размерных
категорий (Σ = 100% годного). Брак — поле, не категория.

## Шорткат «Акт» из зоны 1 — РЕШЕНО (вариант A)
Клик «Акт» на `sent`-партии (зона 1) сначала переводит машину `sent→arrived` (BR-24б), затем открывает
форму акта. Переход живёт в этом действии.

## Срезы

| Срез | Содержание | Проверяемый результат |
|---|---|---|
| **C0** | дизайн формы акта (Claude Design) — **принят** (`acceptance-act-c0.html`) | прототип формы (simple/calibre) |
| **C1** | форма акта + приёмка позиции, БЕЗ склада/денег. **C1a (simple):** вес + брак% → принятый (производный), № акта (BR-9), привязка строки (BR-8), позиция принята, авто-`accepted` машины (BR-13), откат акта (admin). **C1b (calibre):** таблица категорий культуры (% от годного, Σ=100%, дефолт `is_accepted`, ручная привязка нестандарта), двухступенчатый принятый. | партию (simple и calibre) можно принять; статусы и принятый считаются |
| **C2** | движение ингредиента на приёмке позиции (единый `StockMovement kind=ingredient`, фактический вес, фермер→завод), сторно при откате акта, идемпотентность | приёмка двигает склад ингредиентов |
| **C3** | стоимость (принятый × цена строки) + выполнение строк контракта (живой пересчёт, привязки категорий) + зона 3 «Принято» (просмотр актов, откат admin) | видны деньги по партии, контур замкнут |

Порядок ценности: C1a (принять простые — большинство культур) → C1b (калибр: огурцы) → C2 (склад) → C3 (деньги). C1a самостоятелен.

## Зависимости
- C1 ← B4b (вес) + контракты-мини (привязка).
- C2 ← рецептуры (A) + единый складской движок.
- C3 ← схемы калибровки (A5).
- C4 ← контракты-мини.
