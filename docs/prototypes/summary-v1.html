Решения, заложенные в прототип (зафиксированы с пользователем):
  • Охват: текущая неделя (глобальная из URL ?week), ось дни × культуры — ровно «день×культура» из PRD.
  • Метрика ячейки: эффективный вес actual ?? planned (BR-22), Σ по всем поставщикам за день по arrival_date.
  • Итоги по культуре: план (weekTarget для недельной цели или Σ дневных целей), факт (Σ эфф. за неделю), % = факт/план, Δ = факт − план.
  • week-mode культура (цель date=null, BR-23): дневных ячеек нет — факт только в итогах; помечена бейджем «неделя».
  • Подвал «Итого по дням» — только day-mode культуры (BR-23); колонка «Факт» итога = все культуры (вкл. week-mode).
  • Цвета — из Culture.color; интенсивность — внутри культуры (по строке): opacity ∝ значение / макс. ячейка ЭТОЙ культуры за неделю, с минимальным видимым порогом (~18%). Культуры с разными объёмами (томаты 60–80 т/день vs перец 20–50 т/неделю) читаются одинаково; межкультурное сравнение даёт зона итогов, не заливка.
  • Данные: переиспользуется существующий getPlanWeek (server/plan/board.ts) — без новой агрегации и без миграции. Вид read-only, все роли (R).
