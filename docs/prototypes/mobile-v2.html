Мобильные обзорные вьюхи — План недели и Сводка

Продолжение мобильной v1: две read-only обзорные вьюхи на телефоне. План недели (десктоп «План») и Сводка (десктоп «Сводка») питаются от одного загрузчика getPlanWeek — культуры × рабочие дни, эффективный вес (BR-22), недельный/дневной режим цели (BR-23). Плотная матрица культура × день не влезает в 390px → карточный список по культурам. Каркас, таб-бар и drawer наследуются из v1. Правка целей остаётся десктопной. Токены — DESIGN-SYSTEM.md (Geist / Geist Mono, tabular-nums, hairline, radius 6–12, цвета культур из БД), тач-цели ≥44px.

0 · Навигация — как попасть на обзор

Сводка — переключатель вида на мобильной Ленте (сегмент «Лента | Сводка», как десктоп /shipments). План — через drawer, пункт «Планировщик» (на мобиле открывает read-only «План»); «Доска» — десктоп-функция, приглушена. Нижний таб-бар v1 (Лента · Приёмка · Водители · Ещё) не переизобретаем: Сводка живёт под Лентой, План — под «Ещё».

0AСегмент «Лента | Сводка»
На мобильной Ленте — тот же сегмент вида, что на десктопе. Тап по «Сводка» переключает ленту машин на heatmap-обзор (фрейм 2A) без смены раздела.

1 · План недели — read-only

Одна культура = одна карточка: цвет-чип + название, строка план / факт / % выполнения (прогресс-бар как на десктоп-«Плане») + Δ отклонение. Разбивка по дням — сворачиваемые мини-полоски. Week-mode культура (недельная цель) помечена, без дневной разбивки (BR-23). Прогресс нейтральный, единственный акцент (ink) — «достигнуто / перелёт». Правка целей — только десктоп.

1AНеделя с целями и фактом
Томаты — частично перевешено (разбивка раскрыта), Огурцы — только плановые (факт 0), Перец — перелёт (акцент), Патиссоны — week-mode (цель на неделю).

2 · Сводка — read-only

Карточка культуры = чип + название, компактная heatmap-лента по дням (клетки-дни с интенсивностью по объёму — как десктоп-Сводка, но горизонтальной лентой) + итог культуры (факт за неделю). Интенсивность — по строке (внутри культуры), цвет культуры.

2AНеделя с данными — heatmap-ленты
Ленты по культурам: тёмнее = больше тонн в этот день. Патиссоны — week-mode (цель на неделю, без дней). Снизу — итог недели.

Решения / вопросы

Зафиксированное по брифу v2 + открытые вопросы к ревью (PM + пользователь).

Плотная матрица культура×день → карточный список по культурам
решеноНа 390px десктоп-матрица не влезает. Обе вьюхи — одна культура = одна карточка. Тот же код/данные (getPlanWeek), что десктоп — responsive, мобильные варианты.
Обе вьюхи — read-only на мобиле
решеноНикаких инпутов целей/действий — в отличие от десктоп-«Плана». Правка целей остаётся десктопной; шапка помечена «read-only». Эффективный вес (BR-22) и week-mode (BR-23) — те же.
План: отдельный пункт drawer vs. сегмент рядом со Сводкой
вопросСейчас План — пункт drawer «Планировщик» (фрейм 0B), а Сводка — сегмент на Ленте. Альтернатива: сделать План тоже сегментом (Лента | Сводка | План) — одно место для трёх обзоров, но гуще шапка. Что предпочесть?
Сводка: heatmap-лента vs. список чисел
вопросСейчас — горизонтальная heatmap-лента (клетки-дни с интенсивностью). На узком экране 6 клеток — мелковато; альтернатива — простой список «Пн 1,4·Ср 2,0…» без цветовой заливки. Оставить heatmap или дать тумблер?
План: дневная разбивка — свёрнута или раскрыта по умолчанию
вопросВ макете первая карточка раскрыта, остальные — свёрнуты (peek-строка «Пн 1,4·Вт 1,4…»). По умолчанию всё свёрнуто (короче, обзорнее) или всё раскрыто (видны дни сразу)?
