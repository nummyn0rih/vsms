Индикаторы дефицита.

Информационный индикатор нехватки: где годный баланс тары / остаток ингредиента упал ниже порога (AlertRule). Три элемента — панель над матрицей на /packaging и /ingredients + бейдж-счётчик в сайдбаре. Акцент — янтарный, семантика та же, что у индикатора нехватки на доске B5-2. Дефицит ничего не блокирует — только показывает. Всё read-only.

Несколько нарушений — в контексте страницы

2.2
Правило «у любого фермера» — разворот по фермерам
Правило с областью любой фермер разворачивается в строки только по тем фермерам, у кого предмет реально присутствует и баланс ниже порога. Фермеры без этого ингредиента (0 / нет позиции) — не показываются. Провенанс правила виден чипом в шапке группы.

Решения / вопросы

Р1Пустое состояние — скрывать панель

Рекомендация: панель скрыта целиком, когда нарушений нет — здоровая страница без шума. Тихая строка «Дефицита нет» показана как запасной вариант (1.2), если продукту нужна явная отбивка «проверено». Вопрос: нужна ли отбивка, или скрываем молча?

Р2Overflow при 8+ строках

Рекомендация: компактный скролл (≈5 строк) + «Показать все» (1.3). Панель не выдавливает матрицу вниз экрана. Альтернатива: сворачивать в «показать все» сразу от N>5 без скролла. Выберите порог сворачивания.

Р3Сортировка при разных единицах

Сейчас — по абсолютной величине дефицита внутри общего списка (−380 шт выше −24 шт). При смешении тары/бочек и кг/л «величина» несравнима между типами. Вопрос: сортировать глобально по числу, или группировать по позиции, а внутри — по дефициту? (см. 2.2 как группировку.)

Р4Группировка «любой фермер» — плоско или в группы

Правило «любой фермер» даёт несколько строк одной позиции. Базовый вид (2.1) — плоский список, где эти строки просто идут подряд. Группировка с шапкой правила (2.2) — опция для читаемости провенанса. Вопрос: нужен ли режим группировки в v1?

Р5Клик по строке — переход в матрицу

Idea, не обязательна к проработке: клик по строке скроллит/подсвечивает ячейку фермер×тип в матрице ниже. Read-only, никаких действий. Помечаю как кандидата на v2.
