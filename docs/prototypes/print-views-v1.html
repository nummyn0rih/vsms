Печатные формы рабочих видов

Пять A4-листов для распечатки / «Сохранить как PDF»: Отгрузки, Сводка, План недели, Приёмка — компактными таблицами (чёрным по белому, без графики); Аналитика — с графиками (KPI + 4 графика, подогнаны под лист). Отдельные print-представления: чистые таблицы, а не «@media print» поверх интерактивных экранов. Каждый лист печатается своим контекстом (сезон / неделя / фильтры). Токены наследуются из системы — Geist + Geist Mono, tabular-nums, hairline-рамки.

Кнопка «Печать» на каждом листе открывает браузерную печать только этого листа (Ctrl/⌘+P → «Сохранить как PDF»). Экранный хром (кнопки, подписи, блок решений) в печать не попадает.
01

Решения / вопросы

Открыто на ревью печатных форм v1 — до кода.

Отгрузки — за неделю или за весь сезон?
вопросv1 — печать за выбранную неделю (контекст из ленты: W## + фильтры). Сезонный свод машин — отдельный лист (много страниц, подытоги по неделям). Достаточно ли недельного листа для подшивки, или сразу нужен сезонный вариант с разбивкой по неделям?
Включать ли деньги в Аналитику и Приёмку?
вопросСейчас без денег: Приёмка — факт / брак / принято (кг), Аналитика — тонны / % / рейсы. Сумма к оплате (₽/кг × принято) есть в экранной приёмке, но на печатный лист не вынесена. Нужна ли колонка «Сумма, ₽» в Приёмке и денежный KPI в Аналитике, или печатные формы остаются натуральными (кг/т)?
Отдельная «Печать» на каждом экране vs единый «Центр печати»
склоняемсяv1 — кнопка «Печать» на каждом рабочем экране печатает свой лист в текущем контексте (неделя/сезон/фильтры) — простой и предсказуемый путь. «Центр печати» (выбор вида + периода + пакетная печать нескольких листов) — вероятный v2, когда понадобится печать «всего сразу».
Максимум по строке в Сводке — жирным или тонкой штриховкой?
решеноТолько жирное начертание максимальной ячейки дня в строке — без заливки/heatmap на печати. Штриховку не вводим: на ч/б принтере она грязнит таблицу, а жирность читается всегда.
week-mode культуры в Сводке и Плане
решеноКультуры с недельной целью (date=null, BR-23) — строкой с бейджем «неделя»: дневные ячейки объединены, факт только в итоговых колонках. Подвал «Итого по дням» суммирует лишь day-mode культуры; колонка «Факт» итога — все культуры.

VSMS · инструмент снабженца. Статичный макет печатных форм (v1): пять A4-листов — Отгрузки / Сводка / План / Приёмка компактными таблицами, Аналитика — с графиками (KPI + 4 графика). Общий каркас шапки/подвала, @media print скрывает весь хром. Значения (эфф. вес, принято, брак, %, выполнение) вычисляются на лету и не хранятся; контекст (сезон / неделя / фильтры) — из URL. Печать — браузером (Ctrl/⌘+P → PDF); серверной PDF-библиотеки нет.
