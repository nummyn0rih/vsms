Аналитика по культуре

Drill-down из дашборда /analytics — профиль одной культуры за сезон: как идёт именно она, кто её поставляет и с каким качеством. Read-only, отдельный шарабельный экран (?season=) с селектором культуры и «← к дашборду». Агрегат getCultureAnalytics({season, cultureId}) считает на лету (execution по строкам культуры · брак = actual×brak% · приёмка по неделям · калибр из категорий акта) — ничего не хранится. Серия культуры — её Culture.color; брак — нейтральный янтарь; недели — графит. Год-к-году и деньги-KPI — вне среза. Числа иллюстративные; в проде графики — Recharts.

A0

Ниже: динамика приёмки по неделям (area, серия — цвет культуры, план пунктиром), % брака по неделям (нейтральный янтарь), таблица по поставщикам (ключевой разрез — кто даёт объём и кто брак). Калибр-блок скрыт — культура simple.

Решения / вопросы

Открыто на ревью раскладки v1 — до кода на Recharts / getCultureAnalytics.

Отдельный экран, модалка или боковая панель?
склоняемсяОтдельный экран /analytics/culture/[id]?season=: профиль просторный (KPI + 2–3 графика + таблица поставщиков + калибр), ссылка шарабельна, «← к дашборду» возвращает контекст. Модалка/панель зажали бы таблицу и калибр. Утвердить: не нужен ли лёгкий peek-режим (панель) прямо из графика дашборда?
По поставщикам — таблица или бар?
вопросv1 — таблица (Фермер · принято · выполнение · брак · доля): 4–5 числовых колонок читаются в строку, брак и выполнение рядом видно сразу; доля дана мини-баром внутри строки (гибрид). Чистый бар показал бы только объём, потеряв брак/выполнение. Достаточно ли таблицы, или нужен переключатель «таблица / бар по объёму»?
Накладывать ли план на динамику?
склоняемсяДа — пунктиром плановый темп (норма/неделя из контракта), заливка area — факт цветом культуры. Даёт ответ «идём с опережением / отстаём» без второго графика. Риск: у культур без ровного плана линия вводит в заблуждение — тогда прячем плановую линию (fallback только area).
% брака — по неделям или одно число?
вопросПоказаны оба: A — график по неделям (виден всплеск и когда он был), B — одно число + спарклайн-тренд (компактно, для профилей с калибр-блоком снизу). Для большинства культур брак стабилен — одно число + тренд, вероятно, достаточно; график по неделям — для проблемных. Нужен ли тумблер, или фиксируем одно число?
«Выполнение» у поставщика без строки контракта
решеноПоказываем «—», а не 0% / 100%: объём есть, планового ориентира по этому фермеру нет (контракт на культуру не заведён на него). Итог по культуре считает выполнение от суммарного плана. Так «выполнение» не путается с «принял мало».
Калибр — стек долей или отдельные бары категорий?
склоняемсяСтек 100% (доли категорий одной полосой) + «не в зачёт»/брак штриховкой в конце: сразу читается структура партии и куда «утекает» вес. Отдельные бары дали бы точное сравнение категорий, но потеряли бы «сумму = 100% партии». Только для calibre-культур; у simple блок скрыт.

VSMS · инструмент снабженца. Статичный макет /analytics/culture/[id] (v1) — раскладка, оси и цвета до кода на Recharts. Read-only, агрегаты на лету (getCultureAnalytics), ничего не хранится. Серия культуры — Culture.color; брак — нейтральный янтарь, не алярм-красный; недели — графит. Год-к-году и деньги-KPI — вне этого среза.
