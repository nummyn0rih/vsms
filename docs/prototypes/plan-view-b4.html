План — недельные цели по культурам

Третий вид ленты (Таблица | Heatmap | План). Сетка целей культура × день с тумблером гранулярности на строке, столбцом «Неделя» и прогресс-баром у каждой ячейки. Токены наследуются из ленты и вкладки «Нормы»: радиусы 6 px, hairline #ebebeb, Geist + tabular-nums. Прогресс — нейтраль/ink, один акцент (ink) для «достигнуто / перелёт»; статусные цвета отгрузок не используются.

План · рабочая неделя
01
Матрица недели — цели и прогресс
Вид «План» активен в переключателе (Heatmap по-прежнему disabled). Строки — культуры, столбцы — рабочие дни сезона (лето Пн–Сб) + итог «Неделя». В каждой ячейке: редактируемая цель (т, автосейв по blur) и прогресс-бар. Томаты — частично перевешено, Огурцы — набрано только плановыми (факт 0), Перец — перелёт за цель, Патиссоны — гранулярность «на неделю» (дневные ячейки свёрнуты).

План · гранулярность строки
02
Конверсия «по дням» ⇄ «на неделю» (BR-21)
Цель задаётся в одной гранулярности на пару (неделя, культура) — режимы не смешиваются. Тумблер на строке переключает представление без потери итога: дневные цели суммируются в недельную, а недельная — распределяется по рабочим дням равномерно (дальше правится вручную). Аффорданс подсказывается тултипом на неактивной опции.

План · прогресс-бар · три ситуации
03
Анатомия бара (BR-22)
Прогресс = Σ эффективного веса позиций / цель. Эффективный вес — перевеска (actual), если она есть, иначе плановый. Сплошная часть — факт, штриховка — плановое без перевески, вертикальная риска — цель. Недобор виден как зазор до риски, перелёт — как продолжение бара за риской в акцентном тоне.
Этап планирования
Набрано только плановыми
5,0 / 6,0 т
Перевески ещё нет (факт 0) — весь прогресс плановый, бар целиком штрихованный. До риски остаётся зазор: план пока не покрывает цель.
В процессе
Частично перевешено
5,4 / 6,0 т
Часть позиций перевешена (сплошная), часть остаётся плановой (штрих). Эффективный вес = факт там, где он есть, иначе план. Недобор до цели — небольшой зазор.
Цель пройдена
Перелёт за цель
7,2 / 6,0 т+1,2
Суммарный вес превысил цель — бар продолжается за риской акцентным (ink) тоном, риска тоже становится акцентной. Единственный акцент в домене плана.
План · пустые состояния
04
Неделя без целей, но с отгрузками
Ячейки целей пустые (плейсхолдер «—»), но отгрузки на неделе есть — прогресс-бары показывают факт/план без риски-цели. У ячеек без цели — приглушённая подсказка «цель не задана»: видно фактическое движение, но не к чему его соотнести.

VSMS · инструмент снабженца. Статичный макет вида «План» (B4). Цель — на пару (неделя, культура) в одной гранулярности: по дням или на неделю (BR-20/21).
