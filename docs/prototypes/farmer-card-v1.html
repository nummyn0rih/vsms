Карточка поставщика

Сводный read-only экран по одному фермеру — открывается из строки справочника /reference/farmers. Отдельная страница /reference/farmers/[id], активная вкладка — в URL (?tab=). Решения / вопросы

Зафиксировано на ревью v1 (PM + пользователь) — учтено в этой ревизии.

Группировка отгрузок — по неделям + машина в строке
решеноv1 — плоские строки-позиции по ISO-неделям (W##): дата · № машины · статус · культура · план/факт/принято, машина повторяется на каждой позиции. Сворачивание позиций одной машины — бэклог, не в v1.
Глубина истории отгрузок
решеноv1 — текущий сезон целиком; переключатель сезона — бэклог. Для архивного фермера достаточно «последней отгрузки» в мини-сводке.
«Принято» до приёмки
решеноОставлено: — приёмка для sent/arrived, — для planned. Принятый вес не хранится, считается на лету только после акта.
Выполнение сезона в мини-сводке vs. итог по контрактам
решеноОдин источник (getContractExecution) и одно округление. Расхождения 64/65% быть не должно — шапочный KPI считается из того же агрегата, что и итог вкладки. Исправлено: и шапка, и итог по фермеру = 65%, стоимость = 1,24 млн ₽.
Тара «в пути» — направление и адресность
решеноТранзит показан консистентно с дашбордом /packaging: те же направления и сентинелы — завод → фермер (тара едет к фермеру) и фермер → завод (едет с сырьём обратно), пунктирный чип с иконкой рейса, число в колонке «Всего». Своя логика не вводится.
