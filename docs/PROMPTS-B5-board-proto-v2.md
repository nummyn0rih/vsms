# PROMPTS — B5-board-proto-v2: прототип доски v2 (одно-фермерские карточки + сборка машин)

> Пересмотр прототипа доски под решения ревью. Новый экран-механика (сборка/разборка) → прототип до кода.
> Выход: `docs/prototypes/board-b5-v2.html` + правка DESIGN.md. Код — обновлённый B5-1b/merge после.

## Что меняется против v1
1. Карточка = ОДИН фермер (все его культуры дня — несколько чипов). Никаких «Ким +3».
2. Вся неделя БЕЗ горизонтального скролла (6 рабочих дней влезают на Full HD).
3. Высота карточки — по контенту (без обрезки сырья/тары); скролл колонки при многих карточках.
4. Сборка нескольких одно-фермерских карточек в МАШИНУ + разборка обратно.
5. Drag-матрица: planned/собранная-машина — таскаются; sent — таскается (двигает только прибытие);
   arrived/accepted — locked.

## ПРОМПТ — B5-board-proto-v2 (Claude Design)

```text
Обнови прототип «Доска-планировщик» VSMS (была v1: docs/prototypes/board-b5.html). Сохрани визуальный язык
(Geist/Geist Mono, tabular-nums, hairline #ebebeb, radius 6/8, нейтральный хром, status-fills, цвета культур,
БЕЗ эмодзи/градиентов/pill-кнопок, иконки lucide). Сделай v2 со следующими изменениями. Сохрани как
docs/prototypes/board-b5-v2.html.

КОНТЕКСТ: доска планирует поездки к фермерам и сборку их в машины. Колонки = рабочие дни недели.

1) ОДНО-ФЕРМЕРСКАЯ КАРТОЧКА (основной примитив):
   - Один фермер (имя крупно, без «+N»); под ним — несколько чипов культур этого фермера в этот день
     (подложка цвета культуры ~9%, вес 600, единица muted); итог тары; даты отпр(muted)→приб(600);
     бейдж статуса. Компактная: рассчитана влезать в колонку ~260–280px.
   - Высота — ПО КОНТЕНТУ (без max-height/обрезки). Колонка скроллится вертикально при многих карточках.

2) СОБРАННАЯ МАШИНА (карточка-агрегат):
   - Визуально отличима от одиночной (чуть плотнее рамка/левый акцент). Заголовок — «Машина» + перечисление
     фермеров (Мищенко · Ким · Цой) ИЛИ список мини-строк по фермеру; объединённые чипы культур и общий итог
     тары; один статус/даты на машину.
   - Кнопка/действие «Разобрать» → обратно в одно-фермерские карточки.

3) СБОРКА (ключевая новая механика — показать UX):
   - Режим выбора: на одно-фермерских карточках одного дня — чекбокс/выделение по клику. Выбрано ≥2 →
     плавающая панель действий снизу/сверху колонки: «Собрать в машину (N)». Результат — карточка-машина (п.2).
   - Показать оба состояния статикой: (а) несколько выделенных одиночных карточек + панель «Собрать»;
     (б) уже собранная машина с «Разобрать». Привязка водителя на сборке НЕ обязательна (водитель — только
     при отправке).

4) НЕДЕЛЯ FIT-TO-WIDTH (без горизонтального скролла):
   - Колонки = рабочие дни (например 6: Пн–Сб), грид равными долями (1fr), без фикс. min-width, выталкивающего
     дни. Должны помещаться ВСЕ рабочие дни на Full HD (за вычетом сайдбара ~1660px → ~270px/колонка). Скролл
     по горизонтали — только аварийный fallback на узких экранах. Дополнительно предусмотреть/обозначить
     складной сайдбар на планировщике (больше места доске).

5) DRAG-МАТРИЦА (affordance статикой/тултипом):
   - planned (одиночная) и собранная машина (не отправленная) — хват-курсор, таскаются между днями (прибытие
     = день колонки, отправление = −2 рабочих дня).
   - sent машина — таскается, но помечена, что перенос меняет ТОЛЬКО дату прибытия (отправление фиксировано);
     маленький маркер/тултип «перенос: только прибытие».
   - arrived/accepted — locked (приглушены, замок-иконка, без хвата).
   - drop-зона колонки подсвечивается (как в v1).

6) Тулбар сверху: неделя ← → «Сегодня», переключатель «План | Доска» (это /planner; «Лента | Сводка» — на
   /shipments, тут не показывать). «+ Отгрузка» внизу колонки = создать одно-фермерскую плановую на этот день.
   Слот дефицита тары/ингредиента на карточке — нейтральная заглушка (B5-2).

Свёрстай 1 экран: неделя с 5–6 рабочими днями, в одном дне 4–6 одно-фермерских карточек (часть выделена +
панель «Собрать»), в другом — собранная машина из 3 фермеров с «Разобрать», 1 locked (accepted), 1 sent с
маркером «только прибытие», перелёт плана в полосе прогресса. HTML+CSS (+немного JS для выделения/подсветки
drop-зон; даты/сервер не реализовывать).
```
