# PROMPTS — D4: дашборд «Тара» (реальные остатки)

Финальный срез этапа D. Все плечи на месте: D1 начальные остатки + D2 входящая (фермер→-1→завод) +
D3 доставка (завод→-2→фермер). Баланс теперь ЧЕСТНЫЙ — дашборд показывает реальные остатки, а не поток.

## Что меняется против макета D0 (inventory-tare-d0)
Скелет (матрица, sticky, сегмент Годная/Лом, drill-down drawer) — УТВЕРЖДЁН, оставляем. Меняется модель:
- Фермер больше НЕ «−поток»: показываем **реальный остаток** (начальный + доставлено − отгружено).
- Добавляются строки транзита: **«В пути на завод» (-1)** и **«В пути с завода» (-2)** — Σ движений в
  соответствующем сентинеле по типам (глобально). Детализация «кто/куда» — в drill-down.
- Легенда меняется со «поток vs остаток» на «реальный остаток».
- Футер «Итого получено с фермеров» → **«Итого в системе»** (Σ всех локаций по типу = физический объём
  тары, сохраняется при отсутствии disposal — удобный инвариант-проверка).
- Лом — по-прежнему пусто (операции списания — отдельный срез).

## Срезы

- **D4b** — код: `server/inventory/balances.ts` (`getTareBalances`) + экран `app/(app)/packaging/page.tsx`.

## ПРОМПТ — D4a: дизайн-рефреш дашборда (Claude Design)

```text
Обнови дашборд «Тара» приложения VSMS под реальные остатки. Скелет и стиль — из УТВЕРЖДЁННОГО
docs/prototypes/inventory-tare-d0.html (матрица локация×тип, sticky шапка+первый столбец, выделенная
строка «Завод», сегмент «Годная | Лом», drill-down drawer истории движений, токены Vercel). Сохрани
визуальный язык; меняется только содержательная модель.

ИЗМЕНЕНИЯ:
1. Фермерские строки — теперь РЕАЛЬНЫЙ остаток тары (начальный + доставлено − отгружено), обычно
   положительный. Убери трактовку «−поток». Отрицательное значение допустимо (временный зазор, когда
   товар уехал, а доставка ещё не отмечена) — рисуй со знаком, приглушённо, без драматизма.
2. Добавь две строки транзита ПОД фермерами: «В пути на завод» и «В пути с завода» (по типам тары).
   Стиль — как обычные строки, но приглушённый ярлык локации (это «дорога», не склад). Σ глобально.
3. Легенда: «Завод/Фермер — реальный остаток годной тары; В пути — тара в дороге (ещё не у получателя);
   Лом — состояние, появится с операциями списания.» Убери текст про «поток».
4. Футер: «Итого в системе» по каждому типу = Σ всех строк (завод + фермеры + транзит). Это сохраняемый
   физический объём тары.
5. Drill-down ячейки транзита: в истории показывай, из какого рейса/к какому фермеру тара в пути
   (источник Shipment/MaterialShipment).

ТОКЕНЫ/ПРАВИЛА — как в D0: radius 6/8, hairline #ebebeb, Geist, tabular-nums, нейтральная палитра, без
mesh/pill/hero/эмодзи, иконки lucide. Покажи матрицу: Завод + 3–4 фермера (реальные остатки) + 2 строки
транзита, 2–3 типа тары, drill-down на одной ячейке.

Выдай один HTML — сохраню как docs/prototypes/inventory-tare-d4.html.
```

### Чеклист ревью D4a
1. Фермеры — реальный остаток (не «−поток»); легенда обновлена.
2. Две строки транзита (на завод / с завода) по типам.
3. Футер «Итого в системе» = Σ всех строк; сегмент Годная/Лом сохранён (лом пуст).
4. Drill-down транзита раскрывает источник (рейс/фермер).
