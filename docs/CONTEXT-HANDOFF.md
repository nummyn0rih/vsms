## РОЛИ И ПРОЦЕСС (сохранить)
Пользователь: снабженец консервного завода, junior frontend. Кодит через **Claude Code**,
проектирует через **Claude Design**. Ассистент: контроль процесса, ревью, пишет промпты
для Code и Design, ведёт документацию. Процесс: вертикальные срезы; 1 промпт = 1 проверяемая
единица; чеклист ревью после каждого; схему/доки менять только через обсуждение; промпты пишет
ассистент, пользователь запускает и ревьюит. Ассистент держит экспертную роль (строкой
в начале ответа), критикует конструктивно (не поддакивает), по-русски, структурно, с блоком
«Что делать прямо сейчас». Выданные промпты и обновлённые доки — в /mnt/user-data/outputs/.
История этапов в TASKS задним числом НЕ переписывается (только пометки-указатели).

## ЦЕЛЬ
VSMS — внутреннее веб-приложение управления поставками овощного сырья на консервный завод
(замена Excel). Solo + AI. Ядро — **отгрузка (машина/фура)**, не контракт. ~30 фермеров,
5–10 машин/день, до 5 пользователей (пишет почти всё один админ).

## СТЕК (утверждён, реализован)
Next.js App Router + TS, Prisma, PostgreSQL (**Neon**: pooled DATABASE_URL + direct DIRECT_URL),
Auth.js v5 (credentials, роль в сессии), Tailwind + shadcn/ui, @dnd-kit, Recharts (V2), SheetJS,
Vercel. Без Docker/Redis/микросервисов/GraphQL/S3. context7 для свежей доки библиотек.

## КЛЮЧЕВЫЕ ДОМЕННЫЕ РЕШЕНИЯ
**Три базы веса (ядро):** тара — ПЛАНОВЫЙ; ингредиенты — ФАКТИЧЕСКИЙ (перевеска);
контракт/стоимость — ПРИНЯТЫЙ; брак — статистика от фактического, в выполнение НЕ идёт.
- Сезон июнь→май (BR-17); лето Пн–Сб, зима Пн–Пт (BR-18); ISO-недели; SeasonConfig.workdays Int[].
- Отгрузка: 1 машина=1 водитель (nullable на planned, обязателен на sent); 1–4 позиций; даты ±2
  (одна вводится, вторая авто, обе правятся, BR-12); planned→sent→arrived→accepted (BR-13);
  откат только admin+ChangeLog; planned удаляется физически, sent+ только откат+сторно (BR-19);
  сквозной code (в ленте НЕ показывается).
- **МОДЕЛЬ ТАРЫ (B2.5):** тип тары — атрибут ПОЗИЦИИ (ShipmentItem.packaging_type_id,
  nullable=навал). Культура→разрешённые типы через **CulturePackagingType (m2m, ровно один
  is_default)**. Норма — PackagingNorm по ТРОЙКЕ фермер×культура×тип, avg_unit_weight_kg=НЕТТО
  на единицу. **Единая формула units = ceil(planned_weight / нетто-норма)**; box/barrel развилки
  НЕТ; PackagingType.capacity_kg — справочный брутто, в расчёте НЕ участвует. Нет нормы по тройке
  → переход planned→sent блокируется (частичная ошибка). Навал → движения нет.
  (TASKS B2 описывает ДО-B2.5 формулу — историческая, помечена указателем.)
- Движение тары на planned→sent (тип return, фермер→завод), сторно при откате обратным движением,
  идемпотентно (BR-3, BR-19).
- Контракты-мини: строки (культура, объём, цена, label); живой пересчёт; привязка позиции к строке
  (авто если 1; BR-7 та же культура+фермер; BR-8 NOT NULL при accepted).
- Склад: единый StockMovement (kind packaging|ingredient), локации int (0=завод, Farmer.id),
  баланс=Σ движений (может быть отриц.), дефицит не блокирует. Калибры (A5) — на приёмке (этап C).
- Роли: admin (всё) / operator (чтение+прибытие+акт) / user (чтение).

## UI-СЛОВАРЬ СТАТУСОВ (закреплён)

«Черновик» в DOMAIN BR-19 — смысл «можно удалить», НЕ UI-ярлык. В ленте исправлено (17b).

## ДИЗАЙН — ЦВЕТА СТАТУСОВ (двухуровневая модель, КАНОН: DESIGN-SYSTEM → status-fills)
Система Vercel, app-масштаб: radius 6/8px, form-height 40px (компакт 32), hairline #ebebeb,
Geist + tabular-nums, Geist Mono для №акта/недель. Запрещены mesh/pill-100px/hero-dark/эмодзи-иконки.
На статус — ДВЕ заливки: `zone` (фон левой зоны машины) и `badge` (фон бейджа) + `dot` + `fg`:

| статус | zone | badge | dot | fg (текст) |
|---|---|---|---|---|
| planned | #f5f5f5 | #ededed | #b3b3b3 | #888888 |
| sent | #eaf2ff | #d3e5ff | #0070f3 | #0761d1 |
| arrived | #fff6e3 | #ffefcf | #f5a623 | #ab570a |
| accepted | #ddfff7 | #c7f6ea | #29bc9b | #1d8e75 |

accepted НЕ использует #aaffec (cyan-soft) — частая ошибка в старых доках, исправлено.
Цвета культур — из БД Culture.color; подложка строки позиции = color-mix 9% (… 9%, #fff).

**Лента:** двухзонный леджер. Левая зона фикс. 330px, фон=`zone` статуса. Правая: строки позиций
flex:1 (равная высота), подложка=Culture.color ~9%. Группировка неделя→день (по ПРИБЫТИЮ),
подытоги чипами, пустые дни, прошлые свёрнуты/текущая развёрнута, авто-скролл.
**Утверждённые макеты (приняты как дизайн):** лента B0, модалка водителя, диалог «Отгрузка»,
диалоги статусов (Отправить — предпросмотр тары ПО ФЕРМЕРУ + частичная ошибка «нет нормы»→disabled;
Откатить — обычная primary, НЕ destructive), вкладка «Нормы» 3 режима, тулбар в рабочих состояниях.
