# PROMPTS — B5-merge: сборка/разборка машин (плановые отгрузки)

> Сборка нескольких одно-фермерских плановых в одну машину + разборка обратно. Только плановые → леджер не
> затрагивается (движения склада создаются при отправке). Без миграции. + мелочь: дефолт /planner = «Доска».

## Модель
- `Shipment` = машина (нет поля фермера); фермер — на `ShipmentItem.farmer_id`.
- Одно-фермерская карточка = Shipment, у которого все позиции одного farmer_id.
- Сборка: переназначить `shipment_id` позиций выбранных плановых в ОДНУ машину, удалить опустевшие Shipments.
- Разборка: сгруппировать позиции машины по farmer_id → создать по Shipment на фермера, удалить исходный.
- Только `status="planned"` (плановые не имеют StockMovement → ничего не пересчитывать). Водитель не обязателен.

### Чеклист ревью B5-merge
1. assembleShipments: >=2, все planned, один arrival_date; позиции переназначены в target (первый), опустевшие удалены; logChange/RBAC.
2. disassembleShipment: planned; группировка по farmer_id → новый Shipment на фермера (новый code, driver=null, те же даты); исходный удалён; одно-фермерская → no-op.
3. Merge/disassemble не трогают StockMovement (плановые); балансы не пересчитываются.
4. UI: выбор плановых одного дня → «Собрать»; машина → «Разобрать»; router.refresh; admin клиент+сервер.
5. Дефолт /planner = «Доска».
6. Без миграции; sent+ исключены; регресс dnd/«+»/прогресс/ленты/плана зелёный; code уникален.

## После B5-merge — B5-2 (дефицит) + бэклог

вместимости машины. Бэклог идей (не в B5-2, отдельно): тонкий/overlay-скроллбар в «Плане»; шорткат
«целая машина» (фермер+культура+дни-галочки, вес = глобальный дефолт; per-supplier средний вес — позже);
ОТДЕЛЬНО и с приоритетом — `shipment-revert` (откат продуктовой отгрузки arrived→sent/planned + правка
позиций до акта, для админа).
