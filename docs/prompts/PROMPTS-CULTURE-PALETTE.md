# PROMPTS — culture-palette: CULTURE_PALETTE + swatch-пикер в форме культуры

> Заменить произвольный hex-color-picker при создании/правке культуры на **выбор из курируемой палитры проекта**
> (12 цветов). Константа `CULTURE_PALETTE` — единый источник для пикера + документация в `DESIGN-SYSTEM.md`.
> **Без миграции:** `Culture.color` остаётся hex-строкой; рендеринг (чипы/heatmap/графики/печать) уже читает
> `Culture.color` из БД — не трогаем. Прототип утверждён: `docs/prototypes/culture-palette-v1.html`.

## Финальная палитра (12 цветов — из прототипа)
```
#D4322C Красный · #E8730C Оранжевый · #C99700 Горчичный · #5C8A10 Лаймовый ·

```

## Решения (зафиксированы)
| Развилка | Выбор |
|---|---|
| Источник | Константа `CULTURE_PALETTE` (массив `{ hex, label }`, 12 шт) — новый общий модуль; + палитра в `DESIGN-SYSTEM.md`. |
| Новая культура | Только палитра (12 swatch), поле «свой цвет» СКРЫТО. |
| Правка культуры с off-palette цветом | Показать «свой цвет» (hex-поле) с текущим значением — не ломать существующие; можно переключиться на палитру. |
| Дубликаты цвета | Предупреждать (бейдж «занят: <культура>» на занятых swatch + сноска), НЕ запрещать. |
| Схема/рендер | `Culture.color` = hex (без миграции). Рендеринг везде — из `Culture.color`, без изменений. `CULTURE_PALETTE` — только для пикера + доки. |
| Цвет ≠ культура | Пикер показывает цвета + имя оттенка, без культурных ярлыков. |

   - Сетка 12 swatch (по прототипу): выбранный — обводка + галочка; клик выбирает → пишет hex в поле формы color

     фолбэк) с текущим hex в поле — значение сохраняется как есть; пользователь может выбрать палитровый.
   - FK/валидация: color остаётся обязательным/необязательным как в текущей схеме — НЕ менять правило, только
