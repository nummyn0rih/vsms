# PROMPTS — B5-1b: интерактивная доска v2 (card-модель + dnd + валидация + «+»)

> Привести доску к прототипу v2 (`board-b5-v2.html`) и сделать интерактивной. Без миграции.
> Сборка/разборка машин — отдельный срез B5-merge (здесь только рендер машины + drag/validation/create).

## Что входит
1. Card-модель v2: одно-фермерская карточка (1 фермер) vs машина (>1 фермера, `.mcard`); убрать «Ким +3» и
   избыточную подпись «N фермера».
2. Лэйаут v2: колонки `repeat(N,minmax(0,1fr))` (вся неделя без гор. скролла), высота карточки по контенту,
   скролл колонки.
3. dnd (@dnd-kit) + пересчёт дат + drop-валидация.
4. «+ Отгрузка» в колонке → создать одно-фермерскую плановую на этот день.
5. Сайдбар: сворачивать на «Доске», разворачивать вне её (с уважением ручного тоггла).

## Drop-валидация (утверждена)
- Целевой день (= прибытие) — рабочий день (колонки и так только рабочие).
- **Прошлый день (< сегодня) — дроп ЗАПРЕЩЁН** (для всех перетаскиваемых).
- planned / собранная машина (не отправлена): прибытие = день колонки, отправление = прибытие − 2 **рабочих** дня.
- **sent машина: отправление ФИКСИРОВАНО; меняется только прибытие = день колонки; требование `прибытие > отправление`** → дни ≤ отправления невалидны.
- arrived / accepted — не перетаскиваются (locked).

## После B5-1b — B5-merge → B5-2
B5-merge: режим выбора (≥2 одно-фермерских плановых одного дня) → «Собрать в машину» (свести позиции в один
Shipment) + «Разобрать» (разбить обратно на пер-фермерские плановые); только плановые; водитель не обязателен
(назначается при «Отправлено»). B5-2: индикаторы дефицита тары/ингредиентов (+ опц. предупреждение о
вместимости машины при сборке).
