# DESIGN-спека — печатные формы рабочих видов (для Claude Design)

> Задача Claude Design: статичный HTML-макет **лаконичных печатных форм** пяти рабочих видов в формате **A4**
> с `@media print`. Подход утверждён — **print-CSS, отдельные print-представления** (чистые таблицы, НЕ `@media
> print` поверх интерактивных экранов). Прототип = основа будущего кода. Итог — `docs/prototypes/print-views-v1.html`.
> Дальше PM + пользователь ревьюят.

## Принцип (главное)
Два разных случая:
- **Рабочие данные** (Отгрузки · Сводка · План · Приёмка) = **самое нужное в виде таблицы, без лишней графики**.
  НЕ переносить на печать карточки-леджер, heatmap-заливки, прогресс-бары. Компактные таблицы, чёрным по белому,
  для распечатки/подшивки.
- **Аналитика** = **С ГРАФИКАМИ** (наглядность важнее лаконичности) — print-версия дашборда: KPI + 4 графика,
  подогнанные под A4. Графики НЕ упрощать до таблиц.
Каждый вид печатается своим листом, учитывая текущий контекст (сезон/неделя/фильтры). Кнопка «Печать» на экране →
лист → браузерная печать / «Сохранить как PDF».

## Общий каркас листа (одинаков для всех 5)
- **A4 книжная**, поля ~15мм, Geist / Geist Mono (числа — моно, `tabular-nums`), чёрным по белому, hairline-рамки.
- **Шапка документа:** название вида («Отгрузки», «Сводка», «План недели», «Приёмка», «Аналитика сезона») ·
  сезон · период (неделя `W## · даты` или сезон) · дата печати · применённые фильтры (если есть, одной строкой).
- **Таблица(ы)** — компактные, с итоговой строкой. **Подвал:** «Итого …», при необходимости — № страницы.
- `@media print`: скрыть весь хром приложения (навигация, кнопки, «Печать») — печатается только лист.
- Цвет культуры — максимум маленький чип/квадрат в ячейке имени; никаких заливок строк/ячеек.

## Виды и их таблицы (какие колонки — «самое нужное»)
1. **Отгрузки (Лента) — за неделю (или сезон):**
   Таблица: Дата · № машины · Статус · Культура · Поставщик · План/Факт/Принято (кг). Группировка по дням недели
   (подзаголовки), подытоги дня + итог недели (машин, тонн). Тара — опц. колонкой/строкой.
2. **Сводка — неделя:**
   Таблица культура × рабочий день (эффективный вес, т) + колонки Итог: План · Факт · % · Δ. Строка «Итого по дням»
   + недельный итог. Без heatmap-заливки (только числа; допускается тонкая штриховка/жирность максимума — минимально).
3. **План недели:**
   Таблица культура × день: цель / факт (т) по дням + недельные Цель · Факт · % · Δ. week-mode культуры — строкой
   «цель на неделю». Итог недели.
4. **Приёмка:**
   Таблица позиций: № машины · Культура · Поставщик · Факт (кг) · Брак % · Принято (кг) · № акта. Сгруппировать по
   зоне/статусу (Ожидают перевески / На приёмке / Принято) или по дню. Итоги (принято, средний брак).
5. **Аналитика сезона — С ГРАФИКАМИ (print-версия дашборда):**
   - KPI-полоса: Принято т · Выполнение % · Средний брак % · Рейсов ТК · Осталось ~N машин.
   - 4 графика (как на `/analytics`, подогнанные под A4): выполнение по культурам, динамика приёмки по неделям,
     % брака по культурам, рейсы ТК (овощные/материальные, BR-14). Серии — в цвете (Culture.color; графит для ТК;
     янтарь для брака), оси/подписи читаемые. Скомпоновать под A4 (вертикальный стек или 2×2, чтобы влезло).
   - Графики НЕ заменять таблицами. Recharts (SVG) печатается чисто; при печати скрыть только хром/тултипы.

## Состояния (показать)
- Каждый вид — наполненный лист. Плюс одно «пусто» (напр. пустая неделя) — общий пример достаточно.
- (Опц.) как выглядит `@media print` (скрыт хром) — примечанием/скрин-фреймом.

## Визуальный язык (строго)
- Токены — `DESIGN-SYSTEM.md`, **печатная адаптация:** чёрный текст, hairline-рамки таблиц, Geist; числа/веса/
  № недели/№ акта — Geist Mono `tabular-nums`. Компактно (мелкий, но читаемый шрифт таблиц), строго, «документно».
- В **рабочих** видах (Отгрузки/Сводка/План/Приёмка) ЗАПРЕЩЕНО: статус-заливки строк, heatmap, прогресс-бары,
  графики, эмодзи, тени/градиенты, крупный цвет. Статус — словом (Плановая/Отправлена/Прибыла/Принята), не плашкой.
- В **Аналитике** графики РАЗРЕШЕНЫ и обязательны (в цвете, print-friendly). Это единственный вид с графикой.

## Границы (реализм)
- Read-only снимок; значения (эфф. вес, принято, брак, %, выполнение) — вычисляемые на лету (не хранятся).
- Печать — браузером (Ctrl+P → PDF); серверной PDF-библиотеки нет. Контекст (сезон/неделя/фильтры) — из URL.
- Один лист = один вид за выбранный период; печать «всего сразу» — не в этом макете.

## Ревью v1 (PM + пользователь) — принято, решения зафиксированы
Прототип `print-views-v1.html` утверждён. Решения:
- **Ориентация:** **Сводка и План — A4 landscape** (матрица культура×день не влезает в портрет — «1,4/1,5»
  переносится); Отгрузки, Приёмка, Аналитика — portrait. Реализация — per-page `@page` (именованные:
  `@page landscape { size: A4 landscape }` + `page:` на нужных листах).
- **Отгрузки** — за выбранную неделю (контекст ленты: неделя + фильтры); сезонный свод — бэклог.
- **Печать** — кнопка «Печать» на каждом рабочем экране (печатает свой лист в текущем контексте). «Центр печати»
  (выбор вида/периода, пакет) — v2.
- **Деньги — НЕ выводить:** печатные формы в натуральных единицах (кг/т/%/рейсы/брак). ₽ — контрактная сторона,
  отдельная форма позже. (Ни колонки «Сумма ₽» в Приёмке, ни денежного KPI в Аналитике.)
- **Сводка:** максимум дня по строке — жирным (без heatmap/штриховки на печати).
- **week-mode** культуры (BR-23) — строкой с бейджем «неделя», подвал «Итого по дням» только day-mode.
- **Аналитика** — с графиками (KPI + 4 графика), portrait, серии в цвете.

**Нарезка кода:** (print-1) общий print-каркас (шапка/подвал/`@page`/кнопка) + Отгрузки + Приёмка (portrait);
(print-2) Сводка + План (landscape, из `getPlanWeek`); (print-3) Аналитика (portrait, KPI + графики). Каждый —
отдельная `PROMPTS-PRINT-*.md`.
Внизу — блок «Решения/вопросы» (напр.: Отгрузки за неделю vs весь сезон; включать ли стоимость/деньги в Аналитику
и Приёмку; отдельная кнопка «Печать» на каждом экране vs единый «Центр печати» с выбором вида и периода).
