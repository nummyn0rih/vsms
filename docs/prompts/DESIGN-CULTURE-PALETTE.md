# DESIGN-спека — палитра цветов культур (для Claude Design)

> Задача Claude Design: (1) подобрать **палитру цветов культур проекта** и (2) сделать макет **swatch-пикера**
> в форме культуры. Дальше PM + пользователь ревьюят. Итог — `docs/prototypes/culture-palette-v1.html`.
> Это не новый экран, а системный артефакт (палитра) + маленький контрол в существующем диалоге.

## Проблема
Сейчас цвет культуры задаётся **произвольным hex** (color picker) при создании/правке культуры
(`Culture.color`). Цвет культуры торчит ПОВСЮДУ: чипы в ленте/карточках, подложки строк, heatmap «Сводки»
(заливка с opacity), серии графиков аналитики, мобильные карточки, печатные листы. Произвольный hex даёт
неразличимые между собой, «ядовитые» или блёклые цвета, а на печати/в heatmap часть из них разваливается.
Нужна **курируемая палитра проекта**, из которой пользователь выбирает цвет.

## Что подобрать (палитра)
Набор из **10–14 цветов**, пригодных как «цвет культуры». Критерии (все обязательны):
- **Различимы рядом** — 6–8 культур в одном списке/легенде не сливаются (в т.ч. для распространённых типов
  дальтонизма — избегать пар красный/зелёный одинаковой светлоты).
- **Чип на белом** (8–11px квадрат) — читаемый, не блёклый.
- **Заливка heatmap** — работает при низкой прозрачности (18–100%), не превращается в грязь; на насыщенных
  ячейках белый текст читается.
- **Серия графика** (Recharts bar/area) — нормально смотрится сплошной заливкой.
- **Печать** — в цвете выглядит прилично; в ч/б (grayscale) соседние цвета не схлопываются в одинаковый серый.
- Сочетается с системой (`DESIGN-SYSTEM.md`, Vercel-подобная нейтральная база) — насыщенные, но не неоновые.
- Не конфликтует со **статусными** цветами отгрузок (`status-fills`: planned серый · sent синий `#0070f3` ·
  arrived янтарь `#f5a623` · accepted бирюза `#29bc9b`) и с янтарём брака/дефицита — культура не должна
  «читаться» как статус.

Ориентир существующих цветов (из прототипов, можно взять за основу/уточнить):
томат `#d4322c`/`#e5484d`, огурец `#1d8e75`/`#30a46c`, перец `#e8830c`/`#f76b15`, патиссон `#7928ca`/`#8e4ec6`.

Для каждого цвета в макете показать: имя-ярлык, hex, вид как чип, как заливка heatmap (три уровня прозрачности),
как столбец графика. Плюс общий ряд «все цвета вместе» — проверка различимости.

## Что спроектировать (контрол)
**Swatch-пикер в форме культуры** (диалог создания/правки, там же где name / acceptance_type / типы тары):
- Сетка образцов палитры (кликабельные квадраты/кружки), выбранный — с явной отметкой (обводка + галочка).
- **Пометка занятых цветов**: если цвет уже занят другой культурой — подпись/бейдж «занят: Томаты» (не
  запрещать, но предупреждать — дубли мешают различать).
- **Фолбэк «свой цвет»**: свёрнутый пункт с hex-полем — на случай, если нужен цвет вне палитры и для
  существующих культур с off-palette цветом (их цвет показать как «свой», не ломать).
- Состояния: ничего не выбрано (новая культура) · выбран цвет палитры · выбран «свой» цвет · цвет занят другой.

## Границы (реализм)
- `Culture.color` остаётся hex-строкой — **миграции нет**, палитра это набор допустимых значений + UI выбора.
- Существующие культуры с произвольными цветами продолжают работать (показываются как «свой цвет»).
- Палитра должна стать единым источником: попадёт в `DESIGN-SYSTEM.md` + общий константный модуль в коде.

## Референсы
- `docs/DESIGN-SYSTEM.md` (токены, `status-fills`), прототипы `summary-v1.html` (heatmap-заливка),
  `analytics-v1.html` (серии графиков), `lenta-b0.html` (чипы), `print-views-v1.html` (печать, ч/б).
- Форма культуры в коде: `app/(app)/reference/cultures/` (диалог), `server/cultures/`.

## Deliverable
`docs/prototypes/culture-palette-v1.html` — один файл: (1) палитра (образцы с hex + проверки: чип / heatmap
3 уровня / столбец графика / ряд «все вместе» / имитация grayscale-печати); (2) макет swatch-пикера в форме
культуры во всех состояниях. Внизу — блок «Решения/вопросы» (напр.: сколько цветов достаточно; оставлять ли
«свой цвет»; предупреждать или запрещать дубликаты цвета).

## Ревью v1 (PM + пользователь) — решения + ревизия
Прототип `culture-palette-v1.html` — структура принята. Решения:
- **Дубликаты цвета:** предупреждать (бейдж «занят: …»), НЕ запрещать.
- **«Свой цвет»:** для НОВЫХ культур — только палитра (поле «свой цвет» скрыто); поле «свой цвет» показывать
  ТОЛЬКО при правке культуры с off-palette цветом (существующие не ломаем, миграции нет).
- **Единый источник:** палитра → `DESIGN-SYSTEM.md` + константа кода `CULTURE_PALETTE` (чипы/heatmap/графики/печать).
- **Цвет ≠ культура:** в пикере убрать культурные примеры-ярлыки (Томаты/Огурцы) — показывать сетку цветов + имя
  оттенка; культуру↔цвет пользователь распределяет сам.

**Ревизия палитры (нужен новый рендер):**
- **Добавить жёлтый** — сейчас его нет (оранжевый → охра → лайм). Взять **приглушённый золотисто-горчичный**
  (ориентир `#CA8A04`/`#C99700`), НЕ неоновый: должен читаться чипом на белом, работать заливкой heatmap 18–100%,
  не сливаться в grayscale и **быть отличим от статусного янтаря** `#f5a623` («прибыла/брак») — прогнать через все
  проверки прототипа.
- Рассмотреть **замену «Охры»** (`#B7791F`) на этот жёлтый — охра дублирует тёплую-тёмную зону (оранжевый+коричневый).
  Итог: 12 (или 12–13) цветов, обязательно с жёлтым. Финальный набор hex — из ревизии `culture-palette-v1.html`.
