# PROMPTS — E2: UI смешанного рейса (тара + ингредиент)

> Этап E, срез 2/4 — UI. Движок E1 уже принимает смешанные позиции (item_kind packaging|ingredient).
> E2 = расширение УТВЕРЖДЁННОГО экрана «Логистика материалов» (не новый экран): тип груза в строке,
> пикер ингредиента, кол-во с единицей (кг/л), рендер ингредиентных позиций + раздельные итоги.
> Смешанный груз в одном рейсе — штатно.

## Контекст (готово в E1)

- `MaterialShipmentItem(item_kind, packaging_type_id?, ingredient_id?, quantity)` — полиморфизм по item_kind.
- `Ingredient(id, name, unit: IngredientUnit {kg|l}, active)`.
- `schema.ts` — items discriminated union по item_kind (ровно один FK), MAX_ITEMS=4, смешанные kind разрешены.
- `actions.ts` — create/update пишут реальный item_kind + нужный FK; деталь рейса включает ingredient.
- Кол-во: тара = целое (шт), ингредиент = Decimal кг/л.

## Дизайн-решение (УТВЕРЖДЕНО)

Строка позиции: сегмент-тоггл **`Тара | Ингредиент`** слева → свопает пикер (Тип тары ⇄ Ингредиент) и
сбрасывает противоположный FK. Кол-во справа с подсказкой единицы: тара — «шт» (целое), ингредиент —
«кг»/«л» по `Ingredient.unit`. Итоги карточки — раздельно: тара Σ по типам (шт), ингредиенты Σ по единицам
(кг и л отдельно).
