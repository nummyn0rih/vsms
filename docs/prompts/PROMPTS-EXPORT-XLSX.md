# PROMPTS — export-xlsx: экспорт в Excel (Отгрузки + Приёмка)

> Бэклог пользователя. Кнопка **«Экспорт Excel»** на ленте и приёмке — выгрузка данных текущего вида (с активными
> фильтрами) в `.xlsx`. Подход — **клиентский SheetJS** (`xlsx`, в стеке проекта, но ещё не использован). Плоские
> строки (Excel-friendly), без объединённых ячеек. **Без миграции, read-only.** v1 — Отгрузки + Приёмка;
> Сводка/План/Аналитика — бэклог.

## Решения (зафиксированы)
| Развилка | Выбор |
|---|---|
| Библиотека | SheetJS (`xlsx`) — в утверждённом стеке (CLAUDE.md), но ещё не установлена: добавить зависимость. API — свериться через context7 / доки SheetJS. |
| Где генерится | Клиентский экспорт: кнопка строит книгу из данных текущего вида и скачивает файл (`writeFile`). Фильтры ленты/приёмки — клиентские, поэтому берутся как есть. |
| Объём v1 | **Отгрузки** (все недели сезона, прошедшие фильтры; строка = позиция) + **Приёмка** (позиции по фильтрам). Сводка/План/Аналитика — бэклог. |
| Формат строк | ПЛОСКИЕ строки, без merged-ячеек/rowspan (в отличие от печати): № машины/водитель/статус повторяются в каждой строке позиции. Числа — числами (не строками), даты — датой/ISO. |
| Имя файла | Осмысленное: `vsms-отгрузки-2026.xlsx` / `vsms-приёмка-2026.xlsx` (сезон в имени). |

Перед кодом прочитать:

РЕАЛИЗАЦИЯ
1) Общий клиентский хелпер экспорта (напр. lib/xlsx-export.ts): функция(и), принимающие массив плоских объектов
   (строки) + имена колонок + имя листа/файла, строят workbook через SheetJS и вызывают writeFile (скачивание).
   Числовые поля — числами, не строками; заголовки колонок — человекочитаемые. Одна точка использования SheetJS.
2) Лента: кнопка «Экспорт Excel» в тулбаре (рядом с «Печать»). Экспорт — ВСЕ недели сезона, прошедшие текущие
   фильтры (filterFeedWeeks с текущим состоянием), строка = позиция:
   колонки: Неделя(W##) · Дата прибытия · № машины · Водитель · Перевозчик · Статус(словом) · Культура · Поставщик ·
   Тип тары · Кол-во тары · План кг · Факт кг · Принято кг. (Плоско — поля машины повторяются в каждой позиции.)
3) Приёмка: кнопка «Экспорт Excel» в тулбаре. Экспорт — позиции, прошедшие фильтры (machineVisible), строка = позиция:
   колонки: Зона(Прибыли/На приёмке/Принято) · № машины · Водитель · Перевозчик · Культура · Поставщик · Факт кг ·
   Брак % · Принято кг · № акта. Принятый вес — computeAcceptedKg (как на экране/печати), не дублировать формулу.
4) Имя файла: `vsms-отгрузки-<сезон>.xlsx` / `vsms-приёмка-<сезон>.xlsx`.

ОГРАНИЧЕНИЯ

- SheetJS — единственная точка (общий хелпер). Числа числами, не строками. localStorage не использовать.
- Не менять поведение экрана/печати, кроме добавленных кнопок. Доки/TASKS — PM.

- Приёмка: «Экспорт Excel» скачивает .xlsx; позиции по фильтрам; зона/факт/брак/принято/№ акта; принято = computeAcceptedKg.
- Файлы открываются в Excel/LibreOffice, колонки читаемы, суммирование по числовым колонкам работает.
