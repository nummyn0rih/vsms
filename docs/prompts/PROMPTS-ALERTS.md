# PROMPTS — alerts (V1.1): AlertRule-алерты дефицита

> Пороги `AlertRule` уже вводятся (A5, `server/alert-rules`). Дать сравнение порога с реальным балансом →
> индикатор дефицита: **панель «Дефицит» на дашбордах тары/ингредиентов + бейдж-счётчик в меню**. Без
> миграции. Read-only, информационно — дефицит НЕ блокирует (DOMAIN §3, BR-27). Данные балансов —
> из существующих loaders, без дублирования агрегации.

## Решения (зафиксированы с пользователем)
| Развилка | Выбор |
|---|---|
| Место показа | Панель «Дефицит тары» сверху `/packaging`, «Дефицит ингредиентов» сверху `/ingredients`; бейдж-счётчик в сайдбаре на пунктах «Тара»/«Ингредиенты». |
| null-scope («у любого фермера») | Разворачивается по фермерам, ПРИСУТСТВУЮЩИМ в балансах этого предмета (есть ячейка/движения); фермеры без involvement НЕ флагуются (иначе шум «0 у всех»). |
| Состояние | Тара — только `good`-баланс (лом дефицитом не считается). Ингредиент — баланс (у него состояние всегда good). |
| Условие | `balance < threshold` → алерт; `deficit = threshold − balance`. |
| Доступ | Read-only, видимость = как у дашбордов. Никаких мутаций/StockMovement. |

### Чеклист ревью alerts
1. `getActiveAlerts` — балансы из существующих loaders, без дублирования агрегации; чистая функция сравнения выделена.
2. null-scope разворачивается только по присутствующим фермерам (нет шума 0-везде); scope-фермер — точечно.
3. Тара — good-баланс (без scrap); ингредиент — с unit.
4. Панели на /packaging и /ingredients (амбер, скрыты при 0) + бейдж-счётчики в сайдбаре из layout.
5. Без миграции; read-only; дефицит не блокирует; localStorage не используется.
