# DESIGN-спека — аналитика по отдельной культуре (для Claude Design)

> Задача Claude Design: HTML-прототип **drill-down аналитики по одной культуре** за сезон, в стиле дашборда
> `/analytics`. Дальше PM + пользователь ревьюят. Итог — `docs/prototypes/culture-analytics-v1.html`.
> Продолжение дашборда аналитики (Recharts уже введён).

## Контекст
Дашборд `/analytics` даёт СВОДНУЮ картину сезона (KPI + выполнение/динамика/брак/рейсы ТК по всем культурам).
Нужен **разрез по одной культуре** — «профиль культуры в сезоне»: как идёт именно она, кто её поставляет и с
каким качеством. Всё read-only, агрегаты на лету (не хранятся). Год-к-году вне среза (нет истории).

## Доступ / навигация
- Из дашборда: клик по культуре в графике «Выполнение по культурам» (или селектор культуры) → детальный вид.
- Разместить как **отдельный экран** `/analytics/culture/[id]?season=` (простор для профиля, шарабельная ссылка),
  с селектором культуры в шапке и «← к дашборду». (Альтернатива — модалка/панель; вынести в «Решения».)

## Состав (профиль культуры за сезон)
1. **Шапка:** чип+название культуры (цвет из БД), сезон, тип приёмки (по весу / калибр). Селектор культуры + «назад».
2. **KPI-полоса** (по культуре): Принято т / План т · Выполнение % · Средний брак % · Поставок (позиций)/рейсов ·
   **Доля в сезоне** (% принятого этой культуры от всего принятого).
3. **Динамика приёмки по неделям** — line/area, только эта культура (серия — цвет культуры). Опц. наложить план.
4. **% брака** — по неделям (line/bar) ИЛИ одно число + тренд. Нейтральный янтарь.
5. **По поставщикам** (ключевой разрез) — таблица/бар: Фермер · Принято т · Выполнение % (если есть строка
   контракта) · Брак % · доля в культуре. Сортировка по объёму. Показывает, кто даёт объём и кто — брак.
6. **Калибр** (ТОЛЬКО для культур с acceptance_type=calibre): доли категорий (bar/стек) + «не в зачёт»/брак.
   Для simple-культур блок скрыт.

## Состояния (показать)
- Культура с полными данными (simple) — основной вид.
- Культура **calibre** — с блоком калибра.
- Ранняя/скудная культура (мало приёмок) — KPI с «—», графики держат оси / «данных пока нет».

## Визуальный язык (строго)
- Токены/паттерны — как дашборд `/analytics` (`DESIGN-SYSTEM.md`, Recharts, KPI-плитки как `farmer-card`).
- Серия культуры — её цвет (`Culture.color`); брак — нейтральный янтарь; нейтральные метрики (недели) — графит.
- Read-only, без графики ради графики: каждый график отвечает на вопрос (динамика, брак, кто поставляет).
- ЗАПРЕЩЕНО: алярм-красный вне брака, эмодзи-иконки, радуга серий.

## Границы (реализм)
- Данные — новый агрегат `getCultureAnalytics({season, cultureId})`, переиспользующий: execution (строки культуры),
  acceptance (брак = actual×brak% по фермеру/неделе), feed (приёмка по неделям). Ничего не хранить.
- Калибр-разбивка — из категорий акта (BR-10), только для calibre-культур.
- Год-к-году, деньги-KPI — вне этого среза (натуральные единицы; деньги — контрактная сторона).

## Ревью v1 (PM + пользователь) — принято, решения зафиксированы
Прототип `culture-analytics-v1.html` утверждён. Решения:
- **Отдельный экран** `/analytics/culture/[id]?season=` (селектор культуры + «← к дашборду»); peek-панель — v2.
- **По поставщикам — таблица-гибрид** (Фермер · Принято т · Выполнение % · Брак % · Доля мини-баром); без тумблера бар/таблица.
- **Динамика приёмки** — area цветом культуры + **план пунктиром**; если у культуры нет ровного планового темпа — плановую линию прятать (только area).
- **% брака — график ПО НЕДЕЛЯМ (bar)** как единый вид (среднее — уже в KPI); вариант «одно число+спарклайн» убран.
- **Выполнение** у фермера без строки контракта по культуре — «—» (не 0/100%); итог по культуре — от суммарного плана.
- **Калибр** — стек 100% долей категорий + «не в зачёт»/брак штриховкой; только `acceptance_type=calibre`, у simple скрыт.
- Скудная культура — KPI «—», графики держат оси / «данных пока нет», таблица показывает принятое.

## Deliverable
`docs/prototypes/culture-analytics-v1.html` — один файл: профиль культуры (шапка+селектор · KPI · динамика ·
брак · по поставщикам · калибр) в вариантах simple и calibre + скудное состояние. Внизу — блок «Решения/вопросы»
(напр.: отдельный экран vs модалка/панель; нужен ли блок «по поставщикам» таблицей или баром; накладывать ли план
на динамику; % брака по неделям vs одно число).
