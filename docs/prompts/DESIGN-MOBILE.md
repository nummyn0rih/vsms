# DESIGN-спека — мобильная версия v1 (для Claude Design)

> Задача Claude Design: статичный HTML-прототип мобильных экранов в стиле проекта (токены DESIGN-SYSTEM).
> Дальше PM + пользователь ревьюят. Это прототип ВЁРСТКИ на узком экране (~380px). Итог —
> `docs/prototypes/mobile-v1.html` (все экраны как мобильные фреймы, ширина ~380–390px).

## Контекст и подход
VSMS — инструмент снабженца консервного завода. Десктоп стабилен (лента, приёмка, дашборды — в коде).
Нужна **мобильная версия для работы «в полях»** — 3 экрана. Подход (решён PM): **responsive-first, один код,
Tailwind-брейкпоинты**; плотные экраны (лента = леджер-таблица) на мобиле **переверстываются в карточный
список**, а не ужимаются как таблица. Прототип показывает именно мобильную раскладку (не десктоп в узком окне).

## Объём v1 (зафиксировано с пользователем)
1. **Лента отгрузок — просмотр.** 2. **Приёмка — перевеска/отметка прибытия (оператор).** 3. **Контакты водителей.**
(Планировщик, дашборды, справочники, создание/правка отгрузок — НЕ в мобильном v1.)

## Общий мобильный шелл
- **Верхний app-bar:** гамбургер (открывает drawer-навигацию со всеми разделами), заголовок экрана, справа —
  контекстное действие если нужно. Sticky.
- Навигация — **drawer** (выезжает слева) со списком разделов (как десктоп-сайдбар, но полноэкранный).
  (Опц. идея: нижний таб-бар на 3 полевых экрана — пометь как альтернативу, не обязательную.)
- Контент — одна колонка, крупные тач-цели (**≥44px**), без hover-зависимых действий, без горизонтального скролла.
- Числа/веса — `tabular-nums`; статусные заливки/цвета культур — как на десктопе (DESIGN-SYSTEM, из БД).

## Экран 1 — Лента (мобайл, просмотр)
- Переключатель недели (компактный: ‹ W## ›), «Сегодня». Фильтры — в **bottom-sheet** («Фильтры» кнопка →
  шторка с Поставщик/Сырьё/Статус), не тулбар-в-строку.
- Группировка неделя → день (сворачиваемые), внутри — **карточки машин** (не строки таблицы):
  карточка = статус-бейдж, даты рейса, чипы культур + вес, тара; водитель — строка с иконкой и **tel:**-кнопкой
  (тап-звонок). Плановые/отправленные/прибывшие/принятые — по статус-заливкам.
- Read-only (никаких действий правки на мобиле v1). Состояния: неделя с машинами · пустая неделя.

## Экран 2 — Приёмка (мобайл, оператор)
- Зоны (BR-26) как **вкладки/сегмент** вверху: «Ожидают перевески» · «На приёмке» · «Принято» (счётчики).
- Карточка машины: код, водитель (tel:), позиции. По позиции — **инлайн-ввод факт. веса** (крупное поле,
  `inputmode=numeric`, мобильная цифровая клавиатура), кнопка **«Отметить прибытие»** (для машин без перевески)
  и **«Акт»** на позиции `arrived+` — крупные тач-кнопки. Бейдж «N/M взвешено · принято».
- Диалоги (акт приёмки, выбор даты прибытия acceptance auto-date) — **полноэкранные/bottom-sheet** на мобиле.
- Состояния: зона пустая · частично принятая машина · машина «только патиссоны» (без перевески, кнопка отметки).

## Экран 3 — Контакты водителей (мобайл)
- Поиск по фамилии + фильтр по ТК (bottom-sheet или компактный combobox). Список **карточек водителей**:
  ФИО, ТК, **крупная tel:-кнопка** (тап-звонок), мессенджер/инфо если есть. Тап по карточке — детали (опц.).

## Состояния (показать в прототипе)
- Каждый экран — наполненный вид + одно пустое/краевое состояние.
- Открытый drawer (навигация). Открытый bottom-sheet фильтров. Мобильная клавиатура у поля веса (условно).

## Визуальный язык (строго)
- Токены — `docs/DESIGN-SYSTEM.md`: radius 6/8px, hairline `#ebebeb`, Geist/Geist Mono `tabular-nums`,
  канонические статус-заливки, цвета культур из БД. Тач-цели ≥44px, крупный шрифт полей ввода (≥16px — чтобы
  iOS не зумил). Никаких hover-only, узких кликов, горизонтального скролла таблиц.
- ЗАПРЕЩЕНО: эмодзи-иконки, алярм-красный вне статусов, мелкие тач-цели, десктоп-таблицы «как есть».

## Границы (реализм)
- Мобильный v1 — только просмотр ленты + операции приёмки (перевеска/отметка/акт) + звонки водителям.
  Создание/правка отгрузок, планировщик, дашборды — остаются десктопными (в мобиле v1 их нет).
- Responsive — тот же код/данные; прототип задаёт мобильную раскладку и тач-паттерны.

## Референсы
- Десктоп-прототипы: `lenta-b0.html` (лента, статусы), `farmer-card-v1.html` (карточные паттерны, tel:).
- Код: `app/(app)/shipments` (лента), `app/(app)/acceptance` (зоны/акт/перевеска), `app/(app)/reference/drivers`.
- `DESIGN-SYSTEM.md` — токены; `DESIGN.md` — структура экранов; §8 (мобильная отложена — теперь делаем).

## Ревью v1 (PM + пользователь) — принято, решения зафиксированы
Прототип `mobile-v1.html` утверждён. Решения:
- **Навигация:** нижний **таб-бар** на 3 полевых экрана (Лента · Приёмка · Водители) + вкладка **«Ещё»** →
  drawer с остальными разделами (десктопные приглушены). (Не drawer-only.)
- **Акт приёмки:** единый **полноэкранный** диалог (вес + брак + дата прибытия + к оплате). Выбор даты на
  «Отметить прибытие» (без акта) — отдельный маленький **bottom-sheet** (acceptance auto-date, умный дефолт).
- **Объём:** строго 3 экрана. «План»/«Сводка» на мобиле — НЕ в v1 (v2 при необходимости).
- **Подтверждено:** карточный список (не сжатая таблица); тач-цели ≥44px; поле веса `inputmode=numeric`, ≥16px
  (шрифт 22px); статус-заливки/цвета культур — канон DESIGN-SYSTEM/из БД; acceptance auto-date интегрирован в акт.

**Нарезка кода (срезы):** (1) мобильный шелл (app-bar + таб-бар + drawer) + responsive Лента; (2) responsive
Приёмка (зоны/вес/акт/дата); (3) responsive Контакты водителей. Каждый — отдельная `PROMPTS-MOBILE-*.md`.
