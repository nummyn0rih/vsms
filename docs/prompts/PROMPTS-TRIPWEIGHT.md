# PROMPTS — trip-weight (F/V2): фактический средний вес рейса

> Метрика: фактический средний вес овощного рейса за сезон = Σ факт. веса прибывших рейсов / число рейсов.
> Уточняет KPI **«Осталось ~N машин»** на дашборде `/analytics` (сейчас грубо по плановой норме рейса).
> НЕ новый экран — врезка в существующий агрегат/дашборд. **Без миграции, read-only.**

## Решения (зафиксированы)
| Развилка | Выбор |
|---|---|
| Что считаем | Факт. средний вес **овощного** рейса (Shipment): Σ `actual_weight_kg` позиций / число рейсов с перевеской. Материальные рейсы не участвуют (BR-14, у них нет веса сырья). |
| «Рейс» | Одна машина = один `Shipment` (не позиция). Вес рейса = Σ actual его позиций. |
| Какие рейсы | Прибывшие/принятые (`arrived`/`accepted`), у которых есть ≥1 позиция с `actual_weight_kg`. Машины без перевески (напр. «только патиссоны», не взвешивают) — ИСКЛЮЧАЮТСЯ из среднего (иначе занижают). |
| Где показать | KPI «Осталось ~N машин» на `/analytics`: оценка = недобор / факт. ср. вес рейса (если есть ≥1 взвешенный рейс), иначе fallback — плановая норма. Подпись показывает факт ≈X т/рейс и план ≈Y т/рейс рядом (PRD Экран 4). |
| Доступ | Read-only, вычисляется на лету, не хранится. |

### Чеклист ревью trip-weight
1. Чистая агрегация факт. ср. веса (Σ actual / число взвешенных овощных рейсов); машины без перевески исключены; материальные не входят (BR-14).
2. KPI «Осталось» использует факт. вес при наличии, иначе fallback на плановую норму; accepted/target не пересчитаны заново (берутся из getSeasonAnalytics).
3. Подпись KPI: факт ≈X / план ≈Y т/рейс; токены как в текущем дашборде.
4. Без миграции; read-only; ничего не хранится.
