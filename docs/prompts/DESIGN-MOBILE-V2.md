# DESIGN-спека — мобильные План / Сводка (v2, для Claude Design)

> Задача Claude Design: статичный HTML-прототип мобильных раскладок двух read-only вьюх — **План недели** и
> **Сводка** — в стиле проекта. Дальше PM + пользователь ревьюят. Итог — `docs/prototypes/mobile-v2.html`
> (мобильные фреймы ~390px). Продолжение мобильной версии v1 (шелл/таб-бар/drawer уже есть — переиспользовать).

## Контекст
Мобильная v1 закрыта (Лента-просмотр, Приёмка, Контакты водителей — responsive, карточные списки, нижний
таб-бар + drawer). Теперь **v2** добавляет на телефон две обзорные вьюхи:
- **План недели** (десктоп `/planner` вид «План») — матрица культура×день с целями и прогрессом. На мобиле —
  **read-only** (правка целей остаётся десктопной).
- **Сводка** (десктоп `/shipments` вид «Сводка») — heatmap культура×день + итоги. На мобиле — read-only.
Обе вьюхи в проде питаются от одного загрузчика `getPlanWeek` (культуры × рабочие дни, эффективный вес, цели,
итоги план/факт/%/Δ). На узком экране плотная матрица культура×день не влезает → **карточный список по культурам**.

## Мобильная раскладка (обе вьюхи)
Общий каркас: app-bar + переключатель недели ‹ W## › + «Сегодня» (неделя из URL, как везде). Ниже —
**карточки по культурам** (одна культура = одна карточка):
- **План (мобайл):** карточка культуры = цвет-чип + название; строка **план / факт / % выполнения** (бар) +
  **Δ отклонение**; мелко — разбивка по дням недели (мини-полоски или список «Пн 3,1т · Ср 2,4т…»), свёрнуто/
  раскрываемо. week-mode культура (недельная цель) — помечена, без дневной разбивки (BR-23).
- **Сводка (мобайл):** карточка культуры = чип + название; **компактная строка-heatmap по дням** (клетки-дни
  с интенсивностью по объёму — как десктоп-Сводка, но горизонтальной лентой) + итог культуры (факт за неделю).
  Интенсивность — по строке (внутри культуры), цвет культуры.
Итоговая плашка недели снизу: набрано/цель т · %.

## Навигация / доступ (важно — как попасть на мобиле)
- **Сводка** — переключатель вида на мобильной **Ленте** (сегмент «Лента | Сводка», как десктоп /shipments).
- **План** — через **drawer** (пункт «Планировщик» → на мобиле показывает read-only «План»; «Доска» — не в
  мобиле, приглушена). Уточнить на ревью: план как отдельный пункт или тоже сегмент.
- Нижний таб-бар v1 (Лента/Приёмка/Водители/Ещё) — не переизобретать; Сводка живёт под Лентой, План — под «Ещё».

## Состояния (показать)
- План: неделя с целями и фактом (разные % и Δ, перелёт/недобор); пустая неделя (нет целей/отгрузок).
- Сводка: неделя с данными (heatmap-ленты по культурам); week-mode культура; пустая неделя.
- Read-only: никаких инпутов целей/действий на мобиле (в отличие от десктоп-Плана).

## Визуальный язык (строго)
- Токены — `DESIGN-SYSTEM.md`: Geist/Geist Mono `tabular-nums`, hairline, radius 6–12; цвета культур из БД;
  прогресс/бары — как десктоп «План»/«Сводка» (нейтральные + акцент достигнуто/перелёт). Тач-цели ≥44px.
- Heatmap-интенсивность — по строке культуры (как десктоп-Сводка). Δ: перелёт/недобор — как на десктопе.
- ЗАПРЕЩЕНО: эмодзи-иконки, алярм-красный вне статусов, десктоп-матрица «как есть» на узком экране.

## Границы (реализм)
- Обе вьюхи **read-only** на мобиле; правка целей плана — десктоп. Источник данных — тот же `getPlanWeek`
  (эффективный вес BR-22, week-mode BR-23). Неделя — в URL. Responsive: тот же код/данные, мобильные варианты.

## Ревью v2 (PM + пользователь) — принято, решения зафиксированы
Прототип `mobile-v2.html` утверждён. Решения:
- **Навигация:** Сводка — сегмент «Лента | Сводка» на мобильной Ленте (`/shipments`); План — пункт drawer
  «Планировщик» (read-only, «Доска» приглушена). НЕ объединять в один сегмент (разные роуты /shipments vs /planner).
- **Сводка:** heatmap-лента по дням (интенсивность по строке культуры), без тумблера «список чисел».
- **План:** дневная разбивка по умолчанию СВЁРНУТА (peek-строка), раскрытие по тапу; week-mode — без дней (BR-23).
- **Подтверждено:** карточный список по культурам; обе вьюхи read-only (правка целей — десктоп); источник `getPlanWeek`
  (эффективный вес BR-22, week-mode BR-23); прогресс нейтральный + единственный акцент (ink) достигнуто/перелёт.
- **Нит прототипа:** внешний `<script src="mobile-v2.js">` — в проде неактуально (React); на код не влияет.

**Нарезка кода:** (4) мобильная Сводка (сегмент на Ленте + карточки-heatmap); (5) мобильный План (drawer, RO,
карточки прогресса). Каждый — отдельная `PROMPTS-MOBILE-*.md`.
