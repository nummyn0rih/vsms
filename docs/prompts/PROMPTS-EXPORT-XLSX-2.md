# PROMPTS — export-xlsx-2: экспорт Excel Сводки и Плана

> Продолжение экспорта в Excel. Добавить **«Экспорт Excel»** на видах **Сводка** (/shipments) и **План**
> (/planner) — выгрузка матрицы культура×день (эффективный вес) за неделю в `.xlsx`. Клиентский SheetJS через
> общий `lib/xlsx-export`. **Без миграции, read-only.** Кнопки — в row 1 рядом с «Печать» (не в строке фильтров —
> урок с потерей кнопки). Источник — тот же `getPlanWeek`.

## Решения (зафиксированы)
| Развилка | Выбор |
|---|---|
| Библиотека | Тот же клиентский `lib/xlsx-export` (SheetJS уже введён). Не плодить вторую точку. |
| Данные | `getPlanWeek` (та же, что десктоп `SummaryView`/`PlanView` и печать). Числа — числами; эфф. вес BR-22, week-mode BR-23. |
| Сводка — строки | Строка = культура; колонки: Культура · <рабочие дни: эфф. вес> · План · Факт · % · Δ. Строка итога «по дням» + недельный итог. |
| План — строки | Строка = культура; по каждому рабочему дню ПАРА колонок «<день> факт» и «<день> цель» (числа); + Цель · Факт · % · Δ по неделе. Строка итога. |
| week-mode | Культура с недельной целью (BR-23): дневные ячейки пустые; факт/цель только в недельных колонках. |
| Кнопка | «Экспорт Excel» в row 1 тулбара рядом с «Печать»: Сводка — на /shipments вид «Сводка»; План — на /planner вид «План». |
| Имя файла | `vsms-сводка-<сезон>-W<нед>.xlsx` / `vsms-план-<сезон>-W<нед>.xlsx`. |
