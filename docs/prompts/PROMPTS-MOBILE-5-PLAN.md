# PROMPTS — mobile-5: мобильный План (read-only)

> Последний срез мобильной v2. На узком экране (<md) `/planner` показывает вид «План» **read-only** —
> карточки прогресса по культурам (план/факт/%/Δ + сворачиваемая дневная разбивка). «Доска» (drag) —
> десктоп-функция (на мобиле приглушена). Источник — тот же `getPlanWeek`. **Без миграции.** Прототип:
> `mobile-v2.html` (экран 1 План + drawer 0B). Правка целей — только десктоп.

## Решения (зафиксированы)
| Развилка | Выбор |
|---|---|
| Доступ | Пункт drawer «Планировщик» → на мобиле открывает read-only «План». «Доска» — приглушена (десктоп). |
| Раскладка | Карточка культуры: чип+название + строка **план/факт + прогресс-бар** (перевешено · план-штрих · перелёт-акцент · риска-цель) + **% и Δ**. Дневная разбивка — сворачиваемые мини-полоски, **свёрнута по умолчанию** (peek-строка). week-mode культура — плашка «цель на неделю», без дней (BR-23). |
| Read-only | НИКАКИХ инпутов целей/действий (в отличие от десктоп-PlanView). Шапка помечена «read-only». |
| Данные | Тот же `getPlanWeek`; прогресс/итоги — те же величины (эффективный вес BR-22, week-mode BR-23). Не дублировать агрегацию. |
| Десктоп | `PlanView` (md+) без изменений (правка целей остаётся). |

   getPlanWeek (уже грузится в page). Неделя ?week — как сейчас; «План» — дефолтный (и единственный) мобильный вид планировщика.
2) MobilePlanView: карточки по культурам (rows), по прототипу:
   - шапка: чип(color)+название (+ бейдж «неделя» для mode==="week") + итог культуры: факт / цель т.

3) Пункт drawer «Планировщик» активен на мобиле (ведёт на /planner План). Read-only: без инпутов/тумблеров правки.
