# PROMPTS — acceptance-filters: фильтры в приёмке

> Бэклог пользователя. На `/acceptance` нет фильтров — при большом сезоне зоны длинные. Добавить фильтры
> **Поставщик · Культура** (+ поиск по № машины/водителю) на **готовом общем `FilterCombo`** (`components/filters/`),
> с пересчётом счётчиков зон и пробросом в печатный лист. **Без миграции, read-only** (фильтр — это вид).

## Решения (зафиксированы)
| Развилка | Выбор |
|---|---|
| Оси фильтра | **Поставщик (фермер)** · **Культура** + **поиск** (№ машины / водитель). Зоны (Ожидают перевески · На приёмке · Принято) — это уже структура экрана, отдельным фильтром не дублируем. |
| Компонент | Общий `FilterCombo` (`components/filters/`) — тот же, что лента/материалы. Не плодить свой. |
| Состояние | Клиентское (Set, как в ленте/материалах). Печать — через URL (сериализация, как в ленте). |
| Уровень фильтрации | По МАШИНЕ: машина видна, если есть подходящая позиция; счётчики зон и итоги пересчитываются из видимого набора. Пусто-после-фильтра — состояние. |
| Печать | Кнопка «Печать» на /acceptance тянет текущие фильтры в `/print/acceptance` (как лента). |
| Мобильная приёмка | В этот срез НЕ входит (следующим шагом, паттерн bottom-sheet из мобильной ленты). |
