# DESIGN-спека — печатный акт приёмки (для Claude Design)

> Задача Claude Design: статичный HTML-макет **печатной формы акта приёмки** формата **A4** с `@media print`.
> Подход утверждён — **print-CSS роут** (без библиотек): страница рендерит акт, пользователь печатает / «Сохранить
> как PDF». Прототип = основа будущего кода (не только визуальный референс). Итог — `docs/prototypes/act-print-v1.html`.
> Дальше PM + пользователь ревьюят.

## Контекст и правила
VSMS — учёт поставок овощей на консервный завод. **Приёмка позиционная** (BR-13): единица приёмки =
`ShipmentItem`, каждая позиция = своя партия = свой `AcceptanceAct` (BR-9). Значит **один печатный акт =
одна позиция** (у машины с несколькими культурами — несколько актов). Данные акта существуют (вес, брак,
калибр, принятый вес — вычисляемые). Форма — read-only снимок для печати/подписи.

## Формат
- **A4 книжная**, поля ~15–18мм, чёрным по белому (принтер-дружественно), Geist / Geist Mono (числа, № акта —
  моно, `tabular-nums`). Тонкие hairline-линии/рамки таблиц. Цвет культуры — маленький чип, не заливка листа.
- `@media print`: скрыть весь хром приложения (навигация, кнопки) — печатается только лист акта. На экране —
  тот же лист + кнопка «Печать» (вне печати).

## Состав акта (одна позиция)
1. **Шапка:** «АКТ ПРИЁМКИ» + **№ акта** (`{season_year}-{номер}`, BR-9), дата приёмки (weighed_at / прибытие), сезон.
2. **Стороны:** Поставщик (фермер — название, контакты) · Получатель (завод). 
3. **Рейс:** № машины (`Shipment.code`), водитель + ТК, даты отправления → прибытия.
4. **Позиция/приёмка:**
   - Культура + тип приёмки. 
   - **simple:** факт вес (кг) · брак % · **принятый вес** (кг). 
   - **calibre:** таблица категорий (label · % · кг) + строка «Брак» (% · кг); внизу — **принятый вес** =
     Σ принятых категорий; нестандарт помечен «не в зачёт».
5. **Итоги:** фактический вес · брак (кг и %) · **принято, кг**. (Опц.: строка «к оплате» — цена строки контракта
   × принято — если позиция привязана к строке; на ревью решим, включать ли деньги в акт приёмки.)
6. **Подписи:** «Сдал» (поставщик/водитель) и «Принял» (завод/оператор) — линии для подписи + расшифровка + дата.

## Состояния (показать)
- **simple-акт** (вес + брак% + принято).
- **calibre-акт** (таблица категорий + брак + принято; с нестандартом «не в зачёт»).
- (Опц.) как выглядит `@media print` — можно отдельным примечанием/скрин-фреймом.

## Визуальный язык (строго)
- Токены — `DESIGN-SYSTEM.md`, но **печатная адаптация:** чёрный текст, минимум цвета, hairline-рамки, Geist;
  № акта / веса / номера — Geist Mono `tabular-nums`. Официально-строгий, «документный» вид.
- ЗАПРЕЧЕНО: заливки статусов отгрузок, крупный цвет, эмодзи, тени/градиенты (это печать). Чип культуры — мелкий.

## Границы (реализм)
- Read-only снимок; значения (принятый вес, брак, категории) — вычисляемые на лету из акта (не хранятся).
- Один лист = одна позиция (BR-9/13). Печать нескольких актов машины — вне этого макета (по акту на позицию).
- Печать — через браузер (Ctrl+P → PDF); серверной PDF-библиотеки нет (утверждено).
