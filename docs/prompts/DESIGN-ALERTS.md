# DESIGN-спека — AlertRule-алерты дефицита (для Claude Design)

> Задача Claude Design: собрать статичный HTML-прототип в стиле проекта (как `docs/prototypes/lenta-b0.html`,
> `plan-view-b4.html`, `board-b5-v2.html`) для индикаторов дефицита. Дальше PM + пользователь ревьюят.
> Это НЕ код приложения — только прототип вёрстки/состояний. Итог — файл `docs/prototypes/alerts-v1.html`.

## Контекст
VSMS — внутренний инструмент снабженца консервного завода (замена Excel). Уже есть дашборды балансов:
`/packaging` (тара: матрица локация×тип×состояние) и `/ingredients` (ингредиенты: локация×ингредиент).
Пользователь задаёт пороги дефицита (`AlertRule`): для предмета (тип тары или ингредиент) у конкретного
фермера или «у любого фермера» — порог. Нужен **информационный** индикатор: где баланс упал ниже порога.
Дефицит НИЧЕГО не блокирует — только показывает.

## Что спроектировать (3 элемента)
1. **Панель «Дефицит тары»** — сверху страницы `/packaging`, над матрицей балансов.
2. **Панель «Дефицит ингредиентов»** — сверху страницы `/ingredients` (аналогично, с единицами кг/л).
3. **Бейдж-счётчик** в левом сайдбаре на пунктах меню «Тара» и «Ингредиенты» (число активных нарушений).

## Данные одного алерта
Предмет (название типа тары / ингредиента) · фермер · текущий баланс · порог · дефицит (= порог − баланс).
Для ингредиента — единица (кг/л). Пример строки: «Ящик овощной · Фермер Иванов · 120 / 500 шт · −380».
Ингредиент: «Соль · Фермер Петров · 40 / 150 кг · −110».

## Поведение и состояния (показать в прототипе ВСЕ)
- **Несколько нарушений** (панель тары с 3–5 строками; панель ингредиентов с 2–3, разные единицы).
- **Одно нарушение.**
- **Пусто (дефицита нет):** панель скрыта ИЛИ тихая строка «Дефицита нет» (предложи вариант, рекомендую скрывать).
- **Много нарушений** (8+): поведение overflow — компактный скролл или «показать все».
- **Длинные имена фермеров/типов** — не ломают строку.
- Сортировка строк — по величине дефицита (сначала худшее).
- **Бейдж:** амбер-пилюля с числом рядом с пунктом меню; 0 → бейджа нет.

## Визуальный язык (строго)
- Источник токенов — `docs/DESIGN-SYSTEM.md`; app-масштаб: radius 6px (бейдж/кнопки), 8px (панель/контейнер),
  hairline `#ebebeb`, Geist; числа/веса — `tabular-nums`, технические метки — Geist Mono.
- **Акцент дефицита — АМБЕР, нейтральный, НЕ алярм-красный.** Переиспользовать амбер-индикатор дефицита
  с доски B5-2 (та же семантика «нехватка»): фон/бордер/текст в янтарной гамме статуса `arrived`
  (`zone #fff6e3 / badge #ffefcf / dot #f5a623 / fg #ab570a`). Красный `#d4322c` НЕ использовать — дефицит
  информативен, не ошибка.
- Панель — светлый нейтральный контейнер (hairline-бордер, лёгкая тень стека), заголовок «Дефицит тары»
  с амбер-точкой и счётчиком; строки — компактные, число/порог — Geist Mono tabular-nums, дефицит — амбер fg.
- ЗАПРЕЩЕНО: эмодзи как иконки, алярм-красный, pill-кнопки 100px, градиенты-бенды. Хром нейтральный.

## Границы (важно для реализма прототипа)
- Тара — только годный (`good`) баланс (лом дефицитом не считается).
- «У любого фермера» разворачивается по фермерам, у которых предмет реально присутствует (не показывать
  «0 у всех фермеров подряд»).
- Всё read-only. Опционально показать (но не обязательно) hover/клик по строке как «перейти к фермеру/типу
  в матрице» — пометь как идею, без обязательной проработки.
