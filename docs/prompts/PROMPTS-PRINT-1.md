# PROMPTS — print-1: каркас печати + Отгрузки + Приёмка

> Первый срез печатных форм. Общий **print-каркас** (шапка документа + подвал + `@page` + кнопка «Печать») и
> два листа — **Отгрузки** (за неделю) и **Приёмка** (сезон), обе **portrait**, компактными таблицами, БЕЗ денег.
> Подход — print-CSS, отдельные print-роуты (чистые таблицы, не `@media print` поверх интерактивных экранов).
> **Без миграции, read-only.** Прототип утверждён: `docs/prototypes/print-views-v1.html` (листы 01 и 04 + каркас).

## Решения (зафиксированы)
| Развилка | Выбор |
|---|---|
| Каркас | Общий print-компонент: шапка (название вида · сезон · период неделя/сезон · дата печати · фильтры) + таблица + подвал (Итого · «вид · период · лист 1/1»). `@media print` скрывает весь хром приложения. |
| Роут печати | Отдельные print-роуты БЕЗ сайдбара/нав. **`app/print/...`** (обычный сегмент, URL `/print/shipments`, `/print/acceptance`) с bare-layout, auth-гард. *(NB: route-group `(print)` НЕ годится — не даёт URL-сегмента, `(print)/shipments` → `/shipments` = конфликт с существующим роутом.)* Кнопка «Печать» на экране открывает print-роут в текущем контексте (неделя/сезон/фильтры из URL) → браузерная печать. |
| Ориентация | Отгрузки/Приёмка — **A4 portrait** (Сводка/План будут landscape — print-2). `@page { size: A4 portrait; margin: 0 }`; лист = 210×297мм, поля ~15мм внутри. |
| Отгрузки | За выбранную неделю (контекст ленты). Таблица группируется по дням, подытоги дня + итог недели. Статус — словом (Плановая/Отправлена/Прибыла/Принята), НЕ плашкой. |
| Приёмка | Сезон. Таблица группируется по зоне (Прибыли/На приёмке/Принято). Колонки: № машины·перевозчик · Культура · Поставщик · Факт кг · Брак % · Принято кг · № акта. БЕЗ денег. |
| Данные | Те же loaders: Отгрузки — `getFeed`; Приёмка — `getAcceptanceBoard`/`computeAcceptedKg`. Не дублировать агрегацию. |

### Чеклист ревью print-1
1. Общий `PrintSheet` (шапка/подвал) + print-CSS (`@page` portrait, скрытие хрома); классы по прототипу.
2. Print-роуты в `(print)` без сайдбара; кнопки «Печать» на /shipments и /acceptance открывают их с контекстом (URL).
3. Отгрузки — из getFeed (неделя, группировка по дням, подытоги/итог); Приёмка — из getAcceptanceBoard/computeAcceptedKg (зоны, средний брак взвешенный). Без дублей.
4. Деньги не выводятся; статусы словом; ч/б печатная адаптация; localStorage не используется.
5. Без миграции; read-only; интерактивные экраны не тронуты по поведению.
