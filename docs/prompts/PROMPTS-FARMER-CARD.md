# PROMPTS — farmer-card (V1.1): карточка поставщика

> Сводный read-only экран по одному фермеру: страница `/reference/farmers/[id]` с вкладками. v1 — 4 вкладки
> (Основное+Контакты · Контракты · Отгрузки · Тара/ингредиенты), Качество/Аналитика — disabled «скоро».
> **Без миграции.** Всё вычисляется на лету, агрегаты — из СУЩЕСТВУЮЩИХ loaders (не дублировать формулы).
> Прототип утверждён: `docs/prototypes/farmer-card-v1.html`.

## Решения (зафиксированы с пользователем)
| Развилка | Выбор |
|---|---|
| Формат | Отдельная страница `/reference/farmers/[id]` (не модалка). Переход — из строки списка фермеров. |
| Вкладки v1 | 4 активные + 2 disabled «скоро» (Качество/Аналитика). Активная вкладка — в URL `?tab=` (без localStorage). |
| Выполнение/стоимость | ОДИН источник — `getContractExecution` (тот же, что вкладка «Контракты»). Шапочный KPI считать из него же — расхождений (64/65%) быть не должно. |
| Отгрузки | v1 — плоские строки-позиции, группировка по ISO-неделям; текущий сезон. Сворачивание позиций машины и переключатель сезона — бэклог. |
| «Принято» до приёмки | `— приёмка` (sent/arrived) / `—` (planned); принятый вес — на лету только после акта. |
| Тара «в пути» (per-farmer) | Транзит ЭТОГО фермера НЕЛЬЗЯ взять из `getTareBalances` (сентинелы −1/−2/−3 там общесистемные). Реконструировать из исходных документов: **входящий** (−2/−3, завод/др.фермер → X) = `MaterialShipmentItem(farmer_id=X, arrived_at IS NULL)` Σ по предмету; **исходящий** (−1, X → завод) = `ShipmentItem` фермера в отгрузках `status='sent'`, тара через `calcPackagingUnits`. Инвариант Σ движений НЕ трогать (отдельная выборка поверх полей модели). Ингредиенты по −1 не ходят → исходящий транзит только для тары. |
| Email | Обычный `mailto:` из контактов фермера. НИКАКОЙ обфускации (артефакт cloudflare в прототипе игнорировать). |
| Доступ | Read-only; RBAC — как у страниц справочника фермеров (не расширять). |

     статус позиции — производный (planned/sent/arrived/accepted). Группировка по ISO-неделе (workdays),
     недели по убыванию. Плоские строки-позиции.
   - balances: good/лом-остаток фермера — из getTareBalances/getIngredientBalances (фильтр по локации-
     фермеру); НЕ дублировать формулу Σ движений.

### Чеклист ревью farmer-card
1. `getFarmerCard` — один агрегатор; выполнение/стоимость и KPI из `getContractExecution` (единый источник, нет расхождения 64/65%).
2. Отгрузки — ShipmentItem по farmer_id, группировка ISO-неделя, accepted через `computeAcceptedKg`, «— приёмка» до акта.
3. Балансы good/лом — из существующих loaders (фильтр по фермеру, формула Σ не продублирована), подпись БЕЗ «сезон» (баланс сквозной). Транзит per-farmer — ОТДЕЛЬНОЙ выборкой: входящий = `MaterialShipmentItem(farmer_id, arrived_at IS NULL)`; исходящий = Σ StockMovement плеча отправки (from=farmer, to=−1, kind=packaging) отгрузок в `sent` — ИЗ ЛЕДЖЕРА, не пересчётом нормы. Не из сентинел-агрегата balances.
4. Страница `/reference/farmers/[id]`, вкладка в URL `?tab=`, 4 активные + 2 disabled; ссылка из списка; email mailto без обфускации.
5. Без миграции; read-only; ничего не хранится; localStorage не используется; токены/статус-заливки из DESIGN-SYSTEM.
