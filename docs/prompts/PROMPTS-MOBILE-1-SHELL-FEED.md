# PROMPTS — mobile-1: мобильный шелл + лента (просмотр)

> Первый срез мобильной версии v1. **Responsive-first, один код:** десктоп не менять по поведению; на узком
> экране (<md) — мобильный шелл (app-bar + нижний таб-бар + drawer) и **карточная лента** вместо
> леджер-таблицы. Тот же источник данных (getFeed), чистые хелперы переиспользовать. **Без миграции, read-only.**
> Прототип утверждён: `docs/prototypes/mobile-v1.html` (экран 1 + шелл + drawer + bottom-sheet фильтров).

## Решения (зафиксированы с пользователем)
| Развилка | Выбор |
|---|---|
| Подход | Responsive (Tailwind-брейкпоинты), один код/данные. Плотная лента → карточный список на мобиле, НЕ сжатая таблица. |
| Навигация | Нижний **таб-бар** (Лента · Приёмка · Водители) + вкладка **«Ещё»** → drawer со всеми разделами (десктопные приглушены). Гамбургер в app-bar открывает тот же drawer. |
| Объём среза | Мобильный шелл + **Лента (просмотр)**. Приёмка и Водители — отдельные срезы. |
| Лента на мобиле | Read-only: неделя ‹W##› + «Сегодня», фильтры → bottom-sheet, группы неделя→день (сворачиваемые), карточки машин (статус-заливка, позиции, водитель tel:). Действий правки НЕТ. |
| Состояние | Неделя — в URL (как сейчас), фильтры — клиентские (как в десктоп-ленте). localStorage нельзя. |

### Чеклист ревью mobile-1
1. Responsive: <md — мобильный хром + карточная лента; md+ — сайдбар + таблица без регресса (один код, `md:` классы).
2. Навигация: нижний таб-бар (3 экрана + «Ещё») + drawer из lib/nav; активная вкладка по маршруту; тач ≥44px.
3. Лента-карточки на тех же данных/хелперах (getFeed, weekSummary/daySummary/summarizeCultures) — без дублей; read-only.
4. Фильтры — bottom-sheet на логике FilterCombo; неделя в URL; localStorage не используется; статус-заливки/токены из системы.
5. Без миграции; десктоп не тронут по поведению.
